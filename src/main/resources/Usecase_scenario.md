# Usecase_scenario

## 영업 활동을 관리한다

**Basic Path**

1. 영업 관리자는 웹사이트 상단 메뉴에서 [영업] 메뉴 하위의 [영업 활동 관리] 항목을 클릭한다.
2. 시스템은 영업 활동 관리 화면을 출력한다.
    - 화면 상단에는 조회 조건 입력 영역이 표시된다.
        - 관리 기간: 시작일~종료일 선택 (달력 팝업)
        - 채널 유형: 설계사 / 대리점 드롭다운 선택
    - 조회 조건 영역 하단에는 [조회] 버튼이 배치된다.
    - 화면 하단에는 채널별 영업활동 현황 테이블이 표시된다.
        - 테이블 컬럼: 채널명 / 방문건수 / 계약건수 / 전환율 / 목표달성률
    - 초기 진입 시 당월 기준 전체 채널의 영업활동 현황이 출력된다.
        - 출력 기준: 당월 1일 ~ 오늘 날짜
        - 출력 항목: 채널명 / 방문건수 / 계약건수 / 전환율(계약건수/방문건수 %) / 목표달성률(실적/목표 %)
        - 목표달성률 70% 미만 채널은 해당 셀을 빨간색으로 강조 표시된다.
        - 정렬 기준: 목표달성률 낮은 순으로 기본 정렬
3. 영업 관리자는 조회 조건(관리 기간, 채널 유형)을 입력하고 [조회] 버튼을 클릭한다.
4. 시스템은 조회 조건에 맞는 채널별 영업활동 현황 테이블을 출력한다. (A4)
    - 출력 항목: 채널명 / 채널 유형(설계사/대리점) / 방문건수 / 계약건수 / 전환율(계약건수/방문건수 %) / 목표달성률(실적/목표 %)
    - 조회 결과는 목표달성률 낮은 순으로 정렬되어 출력된다.
    - 조회 결과는 페이지 당 20건씩 출력된다.
    - 목표달성률 70% 미만 채널은 해당 셀을 빨간색으로 강조 표시된다.
5. 영업 관리자는 테이블에서 관리할 채널 항목을 클릭한다.
6. 시스템은 선택한 채널의 상세 활동 내역 패널을 출력한다.
    - 채널명, 채널 유형
    - 활동 실적: 방문건수, 계약건수, 전환율, 목표달성률
    - 패널 우측 상단에는 [개선 지시 등록], [채널 모집], [닫기] 버튼이 배치된다.
7. 영업 관리자는 [개선 지시 등록] 버튼을 클릭한다. (A1, A2)
8. 시스템은 개선 지시 입력 폼을 출력한다.
    - 지시 내용 텍스트 입력란
    - 목표 수정 입력란 (숫자)
    - [저장], [취소] 버튼
9. 영업 관리자는 개선 지시 내용 및 수정 목표를 입력하고 [저장] 버튼을 클릭한다. (A3)
10. 시스템은 "개선 지시가 등록되었습니다."라는 팝업 메시지와 함께 등록 완료 결과(관리번호, 등록일시, 담당채널)를 출력한다. (E1)

**Alternative**

A1) 인원 충당이 필요한 경우

1. 영업 관리자는 [채널 모집] 버튼을 클릭한다.
2. [판매채널을 모집한다](https://www.notion.so/341059fb2d6f80ce92c2cd04ee9340b2?pvs=21) 유스케이스 1번으로 이동한다.

---

A2) [닫기] 버튼을 클릭한 경우

1. 영업 관리자는 패널 우측 상단의 [닫기] 버튼을 클릭한다.
2. 시스템은 상세 활동 내역 패널을 닫고 채널별 영업활동 현황 테이블 화면으로 돌아간다.

---

A3) [취소] 버튼을 클릭한 경우

1. 영업 관리자는 [취소] 버튼을 클릭한다.
2. 시스템은 "작성 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업 관리자는 [확인] 버튼을 클릭한다.
4. 시스템은 개선 지시 입력 폼을 닫고 상세 활동 내역 패널로 돌아간다.

---

A4) 조회 조건에 해당하는 영업활동 데이터가 없는 경우

1. 시스템은 테이블 영역에 "조회 가능한 영업활동 데이터가 없습니다."라는 메시지를 출력한다.
2. 영업 관리자는 조회 조건을 수정한 후 [조회] 버튼을 다시 클릭할 수 있다.

**Exception**

E1) 저장 실패 시

1. 시스템은 "저장에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.

## 판매채널을 모집한다

**Basic Path**

1. [영업 활동을 관리한다](https://www.notion.so/341059fb2d6f80bf8a5fef96b9dbab08?pvs=21)  A1)으로부터 넘어온다.
2. 시스템은 채널 모집 화면을 출력한다.
    - 화면 상단에는 기존 모집 공고 목록 테이블이 표시된다.
        - 테이블 컬럼: 모집번호 / 채널유형 / 모집인원 / 모집기간 / 등록일시
    - 화면 우측 상단에는 [신규 모집 등록], [닫기] 버튼이 배치된다.
3. 영업 관리자는 [신규 모집 등록] 버튼을 클릭한다. (A1)
4. 시스템은 신규 모집 등록 폼을 출력한다.
    - 모집 대상 채널 유형: 설계사 / 대리점 드롭다운 선택, 필수
    - 모집 인원: 숫자 입력란, 필수
    - 모집 기간: 시작일~종료일 선택 (달력 팝업), 필수
    - 모집 조건: 텍스트 입력란 (경력, 자격증 등), 선택
    - [저장], [취소] 버튼
5. 영업 관리자는 모집 정보를 입력하고 [저장] 버튼을 클릭한다. (A2, E1)
6. 시스템은 "모집 공고가 등록되었습니다."라는 팝업 메시지를 출력한다.
7. 영업 관리자는 [확인] 버튼을 클릭한다.
8. 시스템은 등록 완료 결과(모집번호, 등록일시, 채널유형, 모집기간)를 출력하고 [영업 활동을 관리한다](https://www.notion.so/341059fb2d6f80bf8a5fef96b9dbab08?pvs=21) 유스케이스로 복귀한다.

**Alternative**

A1) [닫기] 버튼을 클릭한 경우,

1. 영업 관리자는 [닫기] 버튼을 클릭한다.
2. 시스템은 채널 모집 화면을 닫고 [영업 활동을 관리한다](https://www.notion.so/341059fb2d6f80bf8a5fef96b9dbab08?pvs=21) 유스케이스로 복귀한다.

---

A2) [취소] 버튼을 클릭한 경우,

1. 영업 관리자는 [취소] 버튼을 클릭한다.
2. 시스템은 "작성 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업 관리자는 [확인] 버튼을 클릭한다.
4. 시스템은 모집 등록 폼을 닫고 채널 모집 화면으로 돌아간다.

**Exception**

E1) 필수 항목이 누락된 경우

1. 시스템은 해당 필드에 빨간색 테두리를 표시하고 "필수 항목을 입력해주세요."라는 메시지를 출력한다.
2. 영업 관리자는 누락된 항목을 입력한 후 [저장] 버튼을 클릭한다.

## 판매채널 채용을 심사한다

**Basic Path**

1. 영업 관리자는 웹사이트 상단 메뉴에서 [영업] 메뉴 하위의 [채널 채용] 항목을 클릭한다.
2. 시스템은 채널 채용 화면을 출력한다.
    - 화면 상단에는 조회 조건 입력 영역이 표시된다.
        - 모집 기간: 시작일~종료일 선택 (달력 팝업)
        - 채널 유형: 설계사 / 대리점 드롭다운 선택
        - 심사 상태: 대기 / 승인 / 거절 드롭다운 선택
    - 조회 조건 영역 하단에는 [조회] 버튼이 배치된다.
    - 화면 하단에는 지원자 목록 테이블이 표시된다.
        - 테이블 컬럼: 지원자명 / 채널유형 / 지원일 / 경력 / 자격증 / 심사상태
    - 초기 진입 시 전체 지원자 목록이 최근 지원일 순으로 출력된다.
3. 영업 관리자는 조회 조건을 입력하고 [조회] 버튼을 클릭한다.
4. 시스템은 조회 조건에 맞는 지원자 목록을 출력한다. (A3)
5. 영업 관리자는 지원자 목록에서 심사할 지원자 항목을 클릭한다.
6. 시스템은 선택한 지원자의 상세 정보 패널을 출력한다.
    - 지원자명, 채널 유형, 지원일
    - 경력 사항, 자격증 목록
    - 패널 우측 상단에는 [심사 승인], [심사 거절], [닫기] 버튼이 배치된다.
7. 영업 관리자는 [심사 승인] 버튼을 클릭한다. (A1, A2)
8. 시스템은 "해당 지원자를 승인하시겠습니까?"라는 확인 팝업을 출력한다.
9. 영업 관리자는 [확인] 버튼을 클릭한다.
10. 시스템은 심사 승인 처리 후 인사 담당자에게 채널 등록 요청 알림(지원자명, 채널유형, 승인일시)을 자동 발송하고 완료 결과(승인번호, 승인일시, 지원자명)를 출력한다. (E1, E2)

**Alternative**

A1) 지원자를 거절하는 경우,

1. 영업 관리자는 [심사 거절] 버튼을 클릭한다.
2. 시스템은 거절 사유 입력 팝업을 출력한다.
3. 영업 관리자는 거절 사유를 입력하고 [확인] 버튼을 클릭한다.
4. 시스템은 "해당 지원자가 거절 처리되었습니다."라는 메시지를 출력한다.

---

A2) [닫기] 버튼을 클릭한 경우,

1. 영업 관리자는 [닫기] 버튼을 클릭한다.
2. 시스템은 상세 정보 패널을 닫고 지원자 목록 화면으로 돌아간다.

A3) 조회 조건에 해당하는 지원자가 없는 경우, 시스템은 테이블 영역에 "조회 가능한 지원자가 없습니다."라는 메시지를 출력한다.

**Exception**

E1) 채널 등록 요청 알림 발송 실패 시

1. 시스템은 "알림 발송에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.

---

E2) 채널 등록 처리 실패 시

1. 시스템은 "채널 등록 처리에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.

## 영업조직을 평가한다

**Basic Path**

1. 영업 관리자는 웹사이트 상단 메뉴에서 [영업] 메뉴 하위의 [영업 조직 평가] 항목을 클릭한다.
2. 시스템은 영업 조직 평가 화면을 출력한다.
    - 화면 상단에는 조회 조건 입력 영역이 표시된다.
        - 평가 기간: 시작일~종료일 선택 (달력 팝업)
        - 채널 유형: 설계사 / 대리점 드롭다운 선택
    - 조회 조건 영역 하단에는 [조회] 버튼이 배치된다.
    - 화면 하단에는 채널별 성과 현황 테이블이 표시된다.
        - 테이블 컬럼: 채널명 / 채널 유형 / 매출실적 / 계약건수 / 목표달성률 / 평가등급
    - 초기 진입 시 당월 기준 전체 채널의 성과 현황이 출력된다.
        - 출력 기준: 당월 1일 ~ 오늘 날짜
        - 출력 항목: 채널명 / 채널 유형 / 매출실적 / 계약건수 / 목표달성률 / 평가등급
        - 정렬 기준: 목표달성률 낮은 순으로 기본 정렬
3. 영업 관리자는 조회 조건(평가 기간, 채널 유형)을 입력하고 [조회] 버튼을 클릭한다.
4. 시스템은 조회 조건에 맞는 채널별 성과 현황 테이블을 출력한다. (A4)
    - 출력 항목: 채널명 / 채널 유형 / 매출실적 / 계약건수 / 목표달성률 / 평가등급
    - 조회 결과는 목표달성률 낮은 순으로 정렬되어 출력된다.
    - 조회 결과는 페이지 당 20건씩 출력된다.
5. 영업 관리자는 테이블에서 평가할 채널 항목을 클릭한다.
6. 시스템은 선택한 채널의 성과 상세 정보 패널을 출력한다.
    - 채널명, 채널 유형
    - 성과 실적: 매출실적, 계약건수, 목표달성률
    - 패널 우측 상단에는 [평가 등록], [닫기] 버튼이 배치된다.
7. 영업 관리자는 [평가 등록] 버튼을 클릭한다. (A1)
8. 시스템은 평가 등록 폼을 출력한다.
    - 평가 등급 선택 드롭다운 (S / A / B / C / D), 필수
    - 평가 의견 텍스트 입력란, 선택
    - 성과급 지급하기 버튼
    - [저장], [취소] 버튼
9. 영업 관리자는 평가 등급 및 평가 의견을 입력하고 [저장] 버튼을 클릭한다. (A2, A3)(E1)
10. 시스템은 평가 등록 완료 결과(평가번호, 등록일시, 채널명, 평가등급)를 출력한다.
11. 영업 관리자는 [확인] 버튼을 클릭한다.
12. 시스템은 평가 등록 완료 화면을 닫고 채널별 성과 현황 테이블 화면으로 돌아간다.

**Alternative**

A1) [닫기] 버튼을 클릭한 경우

1. 영업 관리자는 [닫기] 버튼을 클릭한다.

---

A2) [취소] 버튼을 클릭한 경우

1. 영업 관리자는 [취소] 버튼을 클릭한다.
2. 시스템은 "작성 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업 관리자는 [확인] 버튼을 클릭한다.
4. 시스템은 평가 등록 폼을 닫고 성과 상세 정보 패널로 돌아간다.

---

A3) 평가 등급이 S 또는 A인 경우

1. 관리자가 성과급 지급하기 버튼을 클릭한다.
2. [성과급 지급을 요청한다](https://www.notion.so/346059fb2d6f805799e8e4ee85b15e3a?pvs=21) 유스케이스 1번으로 이동한다.
3. [성과급 지급을 요청한다](https://www.notion.so/346059fb2d6f805799e8e4ee85b15e3a?pvs=21) 유스케이스로부터 돌아온다.
4. Basic Path 9번으로 이동한다.

---

A4) 조회 조건에 해당하는 데이터가 없는 경우

1. 시스템은 테이블 영역에 "조회 가능한 데이터가 없습니다."라는 메시지를 출력한다.
2. 영업 관리자는 조회 조건을 수정한 후 [조회] 버튼을 클릭한다.

**Exception**

E1) 필수 항목(평가 등급)이 누락된 경우

1. 시스템은 해당 필드에 빨간색 테두리를 표시하고 "필수 항목을 입력해주세요."라는 메시지를 출력한다.
2. 영업 관리자는 누락된 항목을 입력한 후 [저장] 버튼을 클릭한다.

## 성과급 지급을 요청한다

**Basic Path**

1. [영업조직을 평가한다](https://www.notion.so/341059fb2d6f80e89420f7426a2ea647?pvs=21) 유스케이스 A3) 2번으로부터 넘어온다.
2. 시스템은 성과급 지급 요청 화면을 출력한다.
    - 평가 정보: 평가번호, 채널명, 채널 유형, 평가등급
    - 성과급 산출 기준: 등급별 지급 비율 (S: 150%, A: 120%)
    - 산출된 성과급 금액: 기본급 × 등급별 지급 비율로 자동 산출
    - 요청 사유 텍스트 입력란, 선택
    - [요청 제출], [취소] 버튼
3. 영업 관리자는 성과급 금액을 확인하고 [요청 제출] 버튼을 클릭한다. (A1)
4. 시스템은 "성과급 지급을 요청하시겠습니까?"라는 확인 팝업을 출력한다.
5. 영업 관리자는 [확인] 버튼을 클릭한다.
6. 시스템은 성과급 지급 요청서를 인사 담당자에게 자동 전달하고 요청 완료 결과(요청번호, 요청일시, 대상채널, 요청금액)를 출력한다. (E1)
7. 영업 관리자는 [확인] 버튼을 클릭한다.
8. 시스템은 [영업조직을 평가한다](https://www.notion.so/341059fb2d6f80e89420f7426a2ea647?pvs=21) 유스케이스 A3) 3번으로 복귀한다.

**Alternative**

A1) [취소] 버튼을 클릭한 경우

1. 영업 관리자는 [취소] 버튼을 클릭한다.
2. 시스템은 성과급 지급 화면을 닫고 [영업조직을 평가한다](https://www.notion.so/341059fb2d6f80e89420f7426a2ea647?pvs=21) 로 복귀한다.

**Exception**

E1) 요청 처리 실패 시

1. 시스템은 "요청 처리에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.
2. 영업 관리자는 [재시도] 버튼을 클릭한다.

## 교육 계획안을 작성한다

**Basic Path**

1. 영업교육 담당자는 웹사이트 상단 메뉴에서 [교육] 메뉴 하위의 [교육 계획 작성] 항목을 클릭한다.
2. 시스템은 교육 계획 작성 화면을 출력한다.
    - 화면 상단에는 교육 계획 기본 정보 입력 영역이 표시된다.
        - 교육명: 텍스트 입력란, 필수
        - 교육 기간: 시작일~종료일 선택 (달력 팝업), 필수
        - 채널 유형: 설계사 / 대리점 드롭다운 선택, 필수
        - 교육 대상자 수: 숫자 입력란, 필수
        - 교육 예산: 원 단위 입력란, 필수
    - 화면 하단에는 교육 내용 입력 영역이 표시된다.
        - 교육 목표: 텍스트 입력란, 필수
        - 교육 내용: 텍스트 입력란, 필수
        - 교재 목록: 텍스트 입력란, 선택
    - 화면 우측 상단에는 [임시저장], [승인 요청], [취소] 버튼이 배치된다.
3. 영업교육 담당자는 교육 계획 기본 정보 및 교육 내용을 입력한다.
4. 시스템은 입력된 정보의 필수 항목 누락 및 형식 오류 여부를 실시간으로 검증한다. (E1)
5. 영업교육 담당자는 입력 완료 후 [승인 요청] 버튼을 클릭한다. (A1, A2)
6. 시스템은 "영업 관리자에게 승인을 요청하시겠습니까?"라는 확인 팝업을 출력한다.
7. 영업교육 담당자는 [확인] 버튼을 클릭한다.
8. 시스템은 영업 관리자에게 교육 계획 승인 요청 알림(교육명, 교육기간, 채널유형, 교육예산)을 자동 발송한다.
9. 영업 관리자는 알림을 확인하고 [승인] 버튼을 클릭한다. (A3)
10. 시스템은 승인 완료 결과(계획번호, 승인일시, 교육명)를 영업교육 담당자에게 출력한다.

**Alternative**

A1) [임시저장] 버튼을 클릭한 경우

1. 영업교육 담당자는 [임시저장] 버튼을 클릭한다.
2. 시스템은 입력된 내용을 임시저장하고 "임시저장되었습니다."라는 메시지를 출력한다.

---

A2) [취소] 버튼을 클릭한 경우

1. 영업교육 담당자는 [취소] 버튼을 클릭한다.
2. 시스템은 "작성 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업교육 담당자는 [확인] 버튼을 클릭한다.
4. 시스템은 교육 계획안 작성 화면을 닫고 메인 화면으로 돌아간다.

---

A3) 영업 관리자가 반려하는 경우

1. 영업 관리자는 반려 사유를 입력하고 [반려] 버튼을 클릭한다.
2. 시스템은 반려 사유와 함께 반려 알림을 영업교육 담당자에게 발송한다.
3. 영업교육 담당자는 반려 사유를 확인하고 내용을 수정한 후 Basic Path 5번으로 이어진다.

**Exception**

E1) 필수 항목이 누락된 경우

1. 시스템은 해당 필드에 빨간색 테두리를 표시하고 "필수 항목을 입력해주세요."라는 메시지를 출력한다.
2. 영업교육 담당자는 누락된 항목을 입력한 후 [승인 요청] 버튼을 클릭한다.

---

E2) 승인 요청 알림 발송 실패 시

1. 시스템은 "알림 발송에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.

## 교육 제반을 등록한다

**Basic Path**

1. 영업교육 담당자는 웹사이트 상단 메뉴에서 [교육] 메뉴 하위의 [교육 제반 등록] 항목을 클릭한다.
2. 시스템은 교육 제반 등록 화면을 출력한다.
    - 화면 상단에는 교육 계획안 조회 영역이 표시된다.
        - 교육명 검색 입력란
        - 교육 기간: 시작일~종료일 선택 (달력 팝업)
        - [조회] 버튼
    - 조회 영역 하단에는 승인된 교육 계획안 목록 테이블이 표시된다.
        - 테이블 컬럼: 계획번호 / 교육명 / 교육기간 / 채널유형 / 교육대상자수 / 승인일시
3. 영업교육 담당자는 조회 조건을 입력하고 [조회] 버튼을 클릭한다.
4. 시스템은 조회 조건에 맞는 승인된 교육 계획안 목록을 출력한다. (A3)
5. 영업교육 담당자는 목록에서 등록할 교육 계획안 항목을 클릭한다.
6. 시스템은 선택한 교육 계획안 기반의 제반 등록 화면을 출력한다.
    - 교육 기본 정보: 교육명, 교육기간, 채널유형, 교육대상자수 (자동 입력)
    - 제반 정보 입력 영역
        - 교육 장소: 텍스트 입력란, 필수
        - 강사명: 텍스트 입력란, 필수
        - 교재 준비 현황: 텍스트 입력란, 필수
        - 교육 대상자 명단 첨부: 파일 첨부 버튼, 필수
        - 기타 준비 사항: 텍스트 입력란, 선택
    - 화면 우측 상단에는 [저장], [교육 진행], [취소] 버튼이 배치된다.
7. 영업교육 담당자는 제반 정보를 입력하고 [저장] 버튼을 클릭한다. (A1)
8. 시스템은 입력된 정보의 필수 항목 누락 여부를 검증하고 "등록이 완료되었습니다."라는 저장 완료 팝업을 출력한다. (E1)
9. 영업교육 담당자는 저장 완료 팝업에서 [확인] 버튼을 클릭한다. 
10. 시스템은 등록 완료 결과(등록번호, 등록일시, 교육명, 교육장소)를 출력한다.(A2)

**Alternative**

A1) [취소] 버튼을 클릭한 경우

1. 영업교육 담당자는 [취소] 버튼을 클릭한다.
2. 시스템은 "작성 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업교육 담당자는 [확인] 버튼을 클릭한다.
4. 시스템은 제반 등록 화면을 닫고 교육 계획안 목록 화면으로 돌아간다.

---

A2) 교육 진행이 필요한 경우

1. 영업교육 담당자는 [교육 진행] 버튼을 클릭한다.
2. [교육을 진행한다](https://www.notion.so/341059fb2d6f80d6acc7f1cbffa538a4?pvs=21) 로 이동한다.

---

A3) 조회 조건에 해당하는 교육 계획안이 없는 경우

1. 시스템은 "조회 가능한 교육 계획안이 없습니다."라는 메시지를 출력한다.

**Exception**

E1) 필수 항목이 누락된 경우

1. 시스템은 해당 필드에 빨간색 테두리를 표시하고 "필수 항목을 입력해주세요."라는 메시지를 출력한다.
2. 영업교육 담당자는 누락된 항목을 입력한 후 [저장] 버튼을 클릭한다.

## 교육을 진행한다

**Basic Path**

1. [교육 제반을 등록한다](https://www.notion.so/341059fb2d6f80d1acf3d7c23d32a078?pvs=21) A2) 2번으로부터 넘어온다.
2. 시스템은 교육 진행 화면을 출력한다.
    - 교육 기본 정보: 교육명, 교육기간, 교육장소, 강사명, 채널유형 (자동 입력)
    - 교육 진행 정보 입력 영역
        - 출석 대상자 목록: 교육 대상자 명단 자동 로드
        - 출석 여부: 각 대상자별 출석 / 결석 선택 버튼
        - 교육 진행 메모: 텍스트 입력란, 선택
    - 화면 우측 상단에는 [진행 완료], [취소] 버튼이 배치된다.
3. 영업교육 담당자는 출석 대상자별 출석 여부를 선택한다.
4. 시스템은 출석 현황(출석인원/전체인원)을 실시간으로 출력한다.
5. 영업교육 담당자는 교육 진행 메모를 입력하고 [진행 완료] 버튼을 클릭한다. (A1)
6. 시스템은 "교육을 완료 처리하시겠습니까?"라는 확인 팝업을 출력한다.(E1)
7. 영업교육 담당자는 [확인] 버튼을 클릭한다.
8. 시스템은 교육 완료 처리 후 판매채널에게 교육 수료 알림(교육명, 교육일시, 수료여부)을 자동 발송한다.(E2)
9. 영업교육 담당자는 [확인] 버튼을 클릭한다.
10. 시스템은 교육 진행 완료 결과(완료번호, 완료일시, 교육명, 출석인원)를 출력하고 [교육 제반을 등록한다](https://www.notion.so/341059fb2d6f80d1acf3d7c23d32a078?pvs=21) 유스케이스로 복귀한다.

**Alternative**

A1) [취소] 버튼을 클릭한 경우

1. 영업교육 담당자는 [취소] 버튼을 클릭한다.
2. 시스템은 "진행 중인 내용이 저장되지 않습니다. 취소하시겠습니까?"라는 팝업을 출력한다.
3. 영업교육 담당자는 [확인] 버튼을 클릭한다.
4. 시스템은 교육 진행 화면을 닫고 [교육 제반을 등록한다](https://www.notion.so/341059fb2d6f80d1acf3d7c23d32a078?pvs=21) 유스케이스로 복귀한다.

**Exception**

E1) 교육 완료 처리 실패 시

1. 시스템은 "교육 완료 처리에 실패했습니다. 다시 시도해 주세요."라는 팝업 메시지를 출력한다.
2. 영업교육 담당자는 [재시도] 버튼을 클릭한다.

**Basic Path**

1. 고객은 좌측 네비게이션 바에서 [상담 신청] 메뉴를 클릭한다.
2. 시스템은 상담 신청 화면을 출력한다.
    - 상담 유형 선택란: [방문 상담] 버튼이 기본값으로 활성화된 상태로 표시되며, 버튼 클릭 시 하단에 세로로 [전화 상담] / [온라인 상담] 선택 버튼이 드롭다운 형태로 표시된다.
    - [방문 상담] 선택 시 하단에 표시되는 입력란:
        - 희망 방문일시: 날짜(캘린더 선택) + 시간(드롭다운 선택)
        - 방문 장소: [주소 검색] 버튼 클릭 시 주소 검색 팝업 출력, 주소 선택 후 상세 주소 텍스트 입력란 활성화
    - 연락처 입력란 (000-0000-0000 형식)
    - 상담 내용 입력란 (텍스트)
3. 고객은 방문 상담에 필요한 정보(희망 방문일시, 방문 장소, 연락처, 상담 내용)를 입력한다.(A1, A2)

8. 시스템은 상담 신청 상세 내용(고객명, 상담유형, 희망 방문일시, 방문장소, 연락처, 상담내용)을 출력한다.
9. 판매채널(설계사, 대리점)은 [상담 수락] 버튼을 클릭한다.
10. 시스템은 상담 수락 완료 결과(수락일시, 담당자명, 예정 상담일시)를 고객에게 출력한다.

## 보험상품을 제안한다

**Basic Path**

1. 판매채널(설계사, 대리점)은 보험상품 제안 화면에서 보험상품 목록을 조회한다.
2. 시스템은 보험상품 제안 화면(고객 기본 정보, 면담 내용 요약, 보험상품 목록 테이블)을 출력한다.
    - 고객 기본 정보: 고객명 / 연락처 / 면담일시
    - 면담 내용 요약: 고객 반응 / 후속 조치
    - 보험상품 목록 테이블
        - 컬럼: 상품명 / 보험유형 / 월 보험료 / 보장내용
    - 상단 버튼: [상품 선택] / [닫기]
3. 판매채널(설계사, 대리점)은 보험상품 목록에서 제안할 상품을 선택하고 [상품 선택] 버튼을 클릭한다. (A1, A2)
4. 시스템은 선택한 보험상품의 상세 정보(상품명, 보험유형, 월 보험료, 보장내용, 특약사항)를 출력한다.
5. 판매채널(설계사, 대리점)은 [제안서 발송] 버튼을 클릭한다.
6. 시스템은 고객에게 보험상품 제안서(상품명, 보험유형, 월 보험료, 보장내용)를 발송하고 발송 완료 결과(발송일시, 수신고객명)를 출력한다. (E1)
7. 판매채널(설계사, 대리점)은 [닫기] 버튼을 클릭한다.
8. 시스템은 면담기록 관리 화면으로 돌아간다.

**Alternative**

A1) 판매채널이 보험상품 목록에서 원하는 상품을 찾지 못한 경우

2. 시스템은 검색 조건에 맞는 보험상품 목록을 출력한다.

## 청약서를 작성한다

**Basic Path**

1. 판매채널은 웹사이트에서 [청약서 작성] 메뉴를 클릭한다.
2. 시스템은 청약서 작성 화면(고객명, 상품명, 보험기간, 납입방법, 수익자 입력란)을 출력한다.
3. 판매채널은 고객 기본정보(고객명, 생년월일, 연락처, 주소)를 입력한다.
4. 시스템은 입력된 고객정보를 기반으로 가입 가능한 상품 목록을 출력한다.
5. 판매채널은 상품 목록에서 가입할 상품을 선택하고 계약 정보(보험기간, 납입방법, 납입기간, 수익자)를 입력한다.
6. 시스템은 입력된 정보를 기반으로 청약서 초안(고객정보, 상품명, 보험료, 보험기간, 납입방법), 제출 버튼, 전자서명 요청 버튼을 출력한다.
7. 판매채널은 고객 서면 서명본 스캔파일을 첨부하고 [제출] 버튼을 클릭한다. (A1, E1)
8. 시스템은 첨부 파일 업로드 완료 결과(파일명, 업로드 일시)를 출력한다.
9. 판매채널은 [최종 제출] 버튼을 클릭한다.
10. 시스템은 제출 완료 결과(청약번호, 제출일시, 담당 심사자)를 출력한다.

**Alternative**

A1) 온라인 전자서명인 경우

1. 판매채널은 [전자서명 요청] 버튼을 클릭한다.
2. 시스템은 고객에게 전자서명 요청 링크를 발송한다.
3. 고객은 링크에 접속하여 본인인증 후 전자서명을 완료한다.
4. 시스템은 서명 완료된 청약서를 보험 심사자에게 자동 전달한다.

**Exception**

E1) 첨부 파일 업로드 실패 시

1. 시스템은 "파일 업로드에 실패했습니다. 다시 시도해 주세요."라는 메시지를 출력한다.

## 보험상품을 조회한다

3. 고객은 조회 조건(상품유형, 보험기간, 보험료 범위)을 입력하고 [조회] 버튼을 클릭한다

5. 고객은 보험상품 목록에서 상세 조회할 상품을 클릭한다 (A1, A2)

8. 시스템은 보험상품 목록 화면으로 돌아간다

## 보험을 신청한다

**Basic Path**

1. 고객은 상품 상세 화면에서 [보험 신청] 버튼을 클릭한다.
2. 시스템은 보험 신청 화면(고객정보, 보장내용, 특약 선택, 납입방법 입력란)을 출력한다.
3. 고객은 개인정보(이름, 생년월일, 연락처, 주소)를 입력한다.
4. 시스템은 입력된 정보를 기반으로 보험료 산출 결과(기본보험료, 특약보험료, 총보험료)를 출력한다.
5. 고객은 특약 항목을 선택하고 납입방법(월납/연납)을 선택한다.
6. 시스템은 최종 보험료 및 보장내용 요약(상품명, 보험기간, 총보험료, 선택특약)을 출력한다.
7. 고객은 내용을 확인 후 [신청] 버튼을 클릭한다.
8. 시스템은 본인인증 화면(인증 방법 선택: 휴대폰 인증 / 공동인증서)을 출력한다.
9. 고객은 인증 방법을 선택하고 본인인증을 완료한다.
10. 시스템은 신청 완료 결과(신청번호, 신청일시)를 고객에게 출력하고, 보험 심사자에게 신청 내역(고객정보, 상품명, 보험기간, 납입방법, 선택특약)을 자동 전달한다.

**Alternative**

A1) 고객이 특약을 추가하는 경우

1. 고객은 특약 목록에서 추가할 특약을 선택한다.
2. 시스템은 특약 추가 후 변경된 보험료를 출력한다.
3. Basic Path 7번으로 이동한다.

**Exception**

E1) 본인인증 실패 시

1. 시스템은 "본인인증에 실패했습니다. 다시 시도해 주세요."라는 메시지를 출력한다.

---

E2) 가입 불가 고객인 경우

1. 시스템은 "가입 조건에 맞지 않습니다. 고객센터로 문의해 주세요."라는 메시지를 출력한다.

## 부활을 요청한다

**Basic Path**

1. 고객은 실효된 보험상품 상세 화면에서 [부활 신청] 버튼을 클릭한다
2. 시스템은 부활 신청 화면(실효사유, 실효일자, 미납보험료, 부활가능여부)을 출력한다 (E1)
3. 고객은 부활 신청 정보(연락처, 납입방법)를 입력한다
4. 시스템은 부활 시 납입해야 할 미납보험료 및 이자 산출 결과(미납횟수, 미납금액, 이자금액, 총납입금액)를 출력한다
5. 고객은 납입 방법(카드/계좌이체)을 선택하고 [납입] 버튼을 클릭한다
6. 시스템은 납입 처리 결과(납입금액, 납입일시, 영수증번호)를 출력한다 (E2)
7. 고객은 부활 신청 내용을 확인 후 [부활 신청] 버튼을 클릭한다
8. 시스템은 본인인증 화면(인증 방법 선택: 휴대폰 인증 / 공동인증서)을 출력한다
9. 고객은 인증 방법을 선택하고 본인인증을 완료한다
10. 시스템은 부활 신청 완료 결과(신청번호, 신청일시)를 고객에게 출력하고, 보험 심사자에게 부활 신청 내역(고객정보, 계약번호, 미납보험료 납입내역)을 자동 전달한다

**Alternative**

A1) 부활 가능 기간이 초과된 경우,

1. 시스템은 "부활 가능 기간이 초과되었습니다. 신규 가입을 진행해 주세요."라는 메시지를 출력한다.
2. 고객은 [보험상품을 조회한다](https://www.notion.so/341059fb2d6f801f84f0f8a0fb1376cc?pvs=21) 로 이동한다.

**Exception**

E1) 부활 불가 상품인 경우

1. 시스템은 "해당 상품은 부활 신청이 불가합니다."라는 메시지를 출력한다.

---

E2) 납입 처리 실패 시

## 인수 심사를 한다

**Basic Path**

1. 보험 심사자는 웹사이트에서 [인수 심사] 메뉴를 클릭한다.
2. 시스템은 인수 심사 화면(신청번호, 고객명, 상품명, 신청일시)을 출력한다. (E1)
3. 보험 심사자는 심사 대상 목록에서 심사할 신청 건을 클릭한다.
4. 시스템은 신청 상세 정보(고객정보, 상품명, 보험기간, 납입방법, 특약내역, 신청유형)를 출력한다.
5. 보험 심사자는 고객 정보 및 계약 조건을 검토하고 [심사 시작] 버튼을 클릭한다.
6. 시스템은 자동 심사 결과(인수가능여부, 위험등급, 특이사항)를 출력한다 (A1)
7. 보험 심사자는 심사 결과를 확인하고 최종 심사 결과(승인/거절/조건부승인)를 선택한다.
8. 시스템은 심사 결과 입력 화면(심사결과, 거절사유, 조건부승인조건)을 출력한다.
9. 보험 심사자는 심사 결과 내용을 입력하고 [심사 완료] 버튼을 클릭한다.
10. [심사 결과를 전달한다](https://www.notion.so/341059fb2d6f80ea83c7fc7d729d79e3?pvs=21) 유스케이스 1번으로 이동한다.
11. [심사 결과를 전달한다](https://www.notion.so/341059fb2d6f80ea83c7fc7d729d79e3?pvs=21) 유스케이스 8번에서 넘어온다.
12. 시스템은 “심사가 완료되었습니다.” 메시지와 확인 버튼을 출력한다.
13. 보험 심사자는 [확인] 버튼을 클릭한다.
14. 시스템은 심사 완료 결과(심사번호, 심사일시, 처리결과)를 출력한다.

**Alternative**

A1) 자동 심사로 처리가 불가한 경우,

1. 시스템은 "수동 심사가 필요합니다."라는 메시지와 함께 수동 심사 화면을 출력한다.
2. 보험 심사자는 심사 유형(진단심사 / 특인심사 / 일반심사 / 이미지심사)을 선택한다.
3. 시스템은 선택한 심사 유형에 맞는 심사 항목(심사기준, 필요서류목록)을 출력한다.
4. 보험 심사자는 필요 서류(진단서, 고지서, 계약서 등)를 첨부하고 심사 의견을 입력한다.
5. 시스템은 첨부 서류 및 심사 의견 등록 완료 결과를 출력한다.
6. Basic Path 7번으로 이어진다.

**Exception**

E1) 심사 대기 건이 없는 경우

1. 시스템은 "처리할 심사 건이 없습니다."라는 메시지를 출력한다.

---

E2) 심사 결과 저장 실패 시

1. 시스템은 "심사 결과 저장에 실패했습니다. 다시 시도해 주세요."라는 메시지를 출력한다.

2. 시스템은 심사 결과 전달 화면(심사결과, 승인여부, 조건부승인조건, 거절사유)을 출력하고 판매채널에게 심사 결과 알림(심사번호, 고객명, 상품명, 심사결과)을 자동 발송한다
3. 판매채널은 수신한 알림을 확인하고 [결과 확인] 버튼을 클릭한다
4. 시스템은 심사 결과 상세 정보(고객정보, 상품명, 심사결과, 조건부승인조건, 거절사유)를 출력한다 (A1)
5. 판매채널은 내용을 확인 후 [확인 완료] 버튼을 클릭한다
6. 시스템은 확인 완료 처리 결과(확인일시, 처리번호)를 출력한다
7. 판매채널은 [닫기] 버튼을 클릭한다
8. 시스템은 [인수 심사를 한다](https://www.notion.so/341059fb2d6f80cebe39f0a59596c3b3?pvs=21) 유스케이스 11번으로 이동한다.

**Alternative**

A1) 심사 결과가 조건부 승인인 경우

1. 시스템은 조건부 승인 조건(추가 서류 제출, 보험료 할증 등)을 출력한다.
2. 판매채널은 고객에게 조건부 승인 내용을 안내하고 [안내 완료] 버튼을 클릭한다.
3. Basic Path 4번으로 이어진다.

**Exception**

E1) 알림 발송 실패 시

1. 시스템은 "심사 결과 알림 발송에 실패했습니다. 다시 시도해 주세요."라는 메시지를 출력한다.

## 사고를 접수한다

**Basic Path**

1. 사용자가 화면에서 [사고 접수하기] 버튼을 클릭한다.
2. 시스템은 화면에 [전화 바로 접수] 버튼과 [자동차 사고접수] 버튼을 화면에 출력한다.
3. 사용자는 [자동차 사고접수] 버튼을 클릭한다. (A1)
4. 시스템은 자동차 사고접수 화면을 출력한다.
    - 사고 발생 상황 (사물에 피해가 발생한 경우, 사람이 다친 경우)
    - 차량번호 입력 박스
    - 자동차 소유자 입력 박스 (피보험자명)
    - 휴대폰 번호
    - 현장출동 (현장출동 신청 필요, 사고접수만 진행)
    - 위치 기반 서비스 이용약관 동의 체크 박스
    - 비활성화된 [다음] 버튼
    - [취소] 버튼
5. 사용자는 사물에 피해가 발생한 경우를 선택한다. 현장출동 신청을 하고 필수항목 입력한다. 그리고 동의 후 [다음] 버튼을 클릭한다. (A2, A3) (E1)
6. 시스템은 사용자에게 현장출동 신청이 완료되었음을 알리고 현장출동 서비스 부서에 신청 내역을 출력한다.
    - 신청 시간
    - 사고 위치
    - 사고 상황
    - 사고자 휴대폰 번호
    - 사고 차량 번호
    - [현장 출동] 버튼
7. 현장출동 서비스 부서에 알림이 출력되고 출동 직원을 선정 후 [현장 출발] 버튼을 클릭한다.
8. 시스템은 현장출동 접수 완료 알림을 띄우고 고객에게 현장출동 도착 정보를 출력한다.
    - 고객 위치 (정확한 위치 반영하기 버튼)
    - 도착 예정 시간
    - 도착 예정 차량 번호
    - 출동 직원 이름
    - 출동 직원 전화번호
    - 사고 사진 첨부하기
    - 사고 시 대처요령 정리 박스
    - [출동 요청 취소] 버튼
    - [닫기] 버튼
9. 사용자는 사고 사진을 촬영하여 시스템에 첨부한다. (A4, A5)
10. 시스템은 사용자에게 정상 첨부 되었음을 알린다.
11. 고객은 현장출동 직원이 도착하면 [닫기] 버튼을 클릭한다.
12. 시스템은 메인 화면을 출력한다.

**Alternative**

A1) 앱 접수 대신 '전화 바로 접수'를 선택하는 경우

1. 사용자가 '자동차 사고접수'가 아닌 [전화 바로 접수] 버튼을 클릭한다.
2. 시스템은 스마트폰의 기본 전화 앱을 실행하고, 보험사 콜센터 번호(예: 1588-XXXX)를 자동 세팅하여 발신 대기 상태로 전환한다.

---

A2) 사람이 다친 경우를 선택하는 경우

1. 사용자가 '사고 발생 상황'에서 '사람이 다친 경우' 라디오 버튼을 선택한다.
2. 시스템은 화면 상단에 "인명 피해 발생 시 119에 먼저 신고해 주세요."라는 안내 배너와 [119 바로 걸기] 버튼을 출력하고, 화면 하단에 부상자 정보 입력 영역을 추가로 출력한다.
    - 부상자 수 입력 박스 (숫자, 필수)
    - 부상자별 정보 입력 영역 (부상자 수에 따라 행 추가)
        - 부상자 구분 라디오 버튼 (운전자 / 동승자 / 상대 차량 탑승자 / 보행자), 필수
        - 부상 정도 라디오 버튼 (경상 / 중상 / 사망 의심), 필수
    - 119 신고 여부 라디오 버튼 (신고 완료 / 미신고), 필수
    - 이송 병원명 입력 박스 (선택)
    - 사고 상황 추가 설명 텍스트 박스 (선택)
3. 사용자는 부상자 정보(부상자 수, 부상자 구분, 부상 정도, 119 신고 여부)와 차량번호, 자동차 소유자명, 휴대폰 번호 등 필수 항목을 입력하고 위치 기반 서비스 이용약관에 동의한 뒤 [다음] 버튼을 클릭한다. **(E1, E3)**
4. 시스템은 해당 신청 건을 '인명 피해' 우선 처리 건으로 분류하여 현장출동 서비스 부서에 신청 내역을 전달한다.
    - 신청 시간
    - 사고 위치
    - 사고 상황 (인명 피해, 부상자 수, 부상 정도)
    - 119 신고 여부 및 이송 병원명
    - 사고자 휴대폰 번호
    - 사고 차량 번호
    - 현장 출동 버튼 (긴급 표시)
5. Basic Path 7번으로 이동한다.

---

A3) 현장출동 없이 사고접수만 진행하는 경우

1. 사용자가 '현장출동 신청 필요' 대신 '사고접수만 진행' 라디오 버튼을 선택하고 다음을 클릭한다.
2. 시스템은 현장출동 부서로 데이터를 넘기지 않고, 즉시 "사고 접수가 완료되었습니다."라는 팝업을 출력한다.
3. 사용자가 [확인]을 누른다.
4. Basic Path 12번으로 이동한다.

---

A4) 더 정확한 위치를 전송하는 경우

1. 사용자가 화면에 표시된 내 위치가 틀린 것을 확인하고 [정확한 위치 반영하기] 버튼을 클릭한다.
2. 시스템은 지도 화면, 현재위치 버튼, 주소 검색창, 상세 위치 입력 박스, [위치 확인] 버튼을 출력한다.
3. 사용자가 현재위치 버튼을 클릭하여 현재 주소를 입력하고 상세 위치 입력 박스에 자세한 위치를 입력한 뒤 [위치 확인] 버튼을 클릭한다.
4. 시스템은 수정된 위치 정보를 출동 직원에게 실시간으로 전송하고, Basic Path 8번으로 돌아간다.

---

A5) **상황 종료로 출동을 취소하는 경우**

1. 고객은 [출동 요청 취소] 버튼을 클릭한다.
2. 시스템은 “정말 출동 요청을 취소하시겠습니까?” 메시지와 함께 사유 선택 박스(고객 자체 수습, 타사 현장출동 서비스 이용, 잘못된 요청), [확인] 버튼을 출력한다.
3. 고객은 “타사 현장출동 서비스 이용” 박스를 선택하고 [확인] 버튼을 클릭한다.
4. 시스템은 화면에 “요청 취소가 접수되었습니다.” 메시지를 출력한다.
5. Basic Path 12번으로 이동한다.

**Exception**

**E1) 당사 보험 가입 차량이 아닌 경우 (또는 만기된 경우)**

1. 사용자가 [다음] 버튼을 클릭한다.
2. 시스템은 차량번호와 소유자명을 대조하여 **당사 가입 내역이 없거나 실효/만기된 계약임**을 확인하고 "입력하신 차량번호는 당사 자동차보험 가입 내역이 확인되지 않습니다. 차량번호나 소유자명을 다시 확인해 주세요."라는 에러 팝업을 출력한다.
3. 사용자가 [확인]을 클릭한다.
4. Basic Path 4번으로 되돌아간다.

## 보험금을 요청한다

**Basic Path**

1. 고객이 메인 화면에서 '보험금 청구하기'를 클릭한다.
2. 시스템은 개인정보 수집 및 이용 동의 폼을 화면에 출력한다.
    - 필수 약관 내용, 선택 약관 내용
    - 체크 박스
    - [취소] 버튼
    - 비활성화된 [다음] 버튼을 화면에 출력한다.
3. 고객이 필수 약관 동의 여부를 선택하고 [다음] 버튼을 클릭한다. (A)
4. 시스템은 고객 식별을 위한 본인 인증 방식(휴대폰 인증, 간편인증) 선택화면을 출력한다.
5. 사용자는 간편인증 방식을 선택하여 본인인증한다.
6. 인증을 완료한 시스템은 기본 정보 확인 화면을 출력한다.
    - 수령인 이름 (자동 입력됨)
    - 수령인 주민등록 번호 (자동 입력됨)
    - 수령인 휴대전화번호 (자동 입력되지만 변경 가능)
    - [휴대전화 번호 변경] 버튼
    - 피보험자 선택 박스
    - [취소] 버튼
    - 비활성화된 [다음] 버튼
7. 사용자는 내용 확인 후 [다음] 버튼을 클릭한다. (A)
8. 시스템이 화면에 청구 내용을 입력할 폼을 출력한다.
    - 청구 유형(질병, 재해)
    - 청구 사유(입원, 수술, 통원, 실손 입원, 실손 통원, 기타) - 모두 선택 가능
    - 병명/진단명 선택 박스
    - 비활성화된 [다음] 버튼
    - [취소] 버튼
9. 사용자는 청구 유형, 청구 사유, 진단명을 작성한 뒤 [다음] 버튼을 클릭한다. (A, A2, A3)
10. 시스템은 보험금을 지급받을 계좌 선택 화면을 출력한다.
    - 계좌 선택 (등록된 계좌, 새로운 계좌)
    - 은행/증권
    - 계좌번호
    - 비활성화된 [다음] 버튼
11. 사용자가 은행을 선택하고 계좌번호를 입력한 뒤 [다음] 버튼을 클릭한다. (A, A4)(E1)
12. 시스템은 계좌 인증 성공 안내를 출력하고 보험금 지급심사 안내 방법 선택 화면을 출력한다.
    - 지급내역 안내 (안내톡/문자메시지, 이메일, 우편물(집), 우편물(직장), 신청안함)
    - 수령인 휴대폰 번호 (자동 입력)
    - 진행과정 안내 받기 체크 박스
    - 담당 FP 통지 체크 박스
    - [취소] 버튼
    - 비활성화된 [다음] 버튼
13. 사용자는 지급내역 안내 방식을 선택하고 진행과정 안내 받기를 체크한 뒤 다음 버튼을 클릭한다. (A)
14. 시스템은 필요서류 첨부 화면을 출력한다.
    - 이미지 첨부 버튼 (카메라로 촬영하기, 앨범에서 여러 장 첨부하기, PDF 파일 첨부하기)
    - 필요서류 안내 화면
        
        (입원: 진단서 또는 입퇴원확인서, 수술: 진단서, 수술확인서, 입퇴원확인서 중 택 1, 통원: 진단서, 통원확인서, 진료확인서, 소견서, 진료차트, 처방전 중 택 1, 실손의료비(입원/통원: 진료영수증, 진료비, 세부내역서)
        
    - [취소] 버튼
    - 비활성화된 [다음] 버튼
15. 사용자는 앨범에서 여러 장 첨부하기 버튼을 클릭하여 필요서류를 첨부한 뒤 다음 버튼을 클릭한다. (A)
16. 시스템은 최종 확인 화면을 출력한다.
    - 신청내역
        - 피보험자
        - 청구 이유
        - 첨부 서류
    - 수령정보
        - 수령인
        - 연락처
        - 안내 방법
        - 지급 계좌
    - [취소] 버튼
    - 비활성화된 [신청하기] 버튼
17. 사용자는 이상이 없음을 확인하고 [신청하기] 버튼을 클릭한다. (A)
18. 시스템은 신청 접수 정상 완료 팝업을 띄운다.

**Alternative**

A) 사용자가 [취소] 버튼을 누른 경우

1. 사용자가 [취소] 버튼을 클릭한다.
2. 시스템은 화면에 “이전에 작성한 내용은 모두 사라집니다. 정말 취소하시겠습니까?”라는 안내와 함께 [네], [아니요] 버튼을 출력한다.
3. 사용자는 [네] 버튼을 클릭한다
4. 시스템은 메인 화면을 출력한다.

A1) 사용자가 휴대폰 인증 방식을 선택한 경우

1. 사용자가 휴대폰 인증 방식을 선택한다.
2. 시스템은 휴대폰 인증 화면을 출력한다
    - 이름 입력 박스
    - 통신사 선택 박스 (SKT, KT, LGU+, 알뜰폰 SKT, 알뜰폰 KT, 알뜰폰 LGU+)
    - 전화번호 입력 박스
    - [인증번호 보내기] 버튼
3. 사용자는 이름과 전화번호를 입력하고 통신사를 선택한 뒤 [인증번호 보내기] 버튼을 누른다.
4. 시스템은 인증번호 입력 박스를 출력한다.
5. 사용자는 핸드폰으로 메시지를 받아 인증번호를 입력한다.
6. Basic Path 8번으로 이동한다.

A2) 청구 유형을 재해로 선택할 경우

2. 시스템은 화면에 청구 내용 입력 폼의 구성이 달라진다.
    - 청구사유(질병, 재해)
    - 청구 사유(입원, 수술, 통원, 실손 입원, 실손 통원, 기타)

    - 사고 날짜

    - 비활성화된 [다음] 버튼
3. 사용자는 항목에 값을 입력한 뒤 [다음] 버튼을 클릭한다.
4. Basic Path 12번으로 이동한다.

A3) 청구 사유에 실손 입원이나 실손 통원이 포함되어 있는 경우

3. 사용자는 [아니요]를 체크하고 [다음] 버튼을 클릭한다.
4. Basic Path 12번으로 이동한다.

A4) 보험사에 회원으로 등록되어 있어 등록된 계좌를 선택하는 경우

    - [다음] 버튼

2. 고객은 [확인] 버튼을 누른다.

## 현장 출동 정보를 기록한다

**Basic Path**

1. 현장출동 직원이 화면에서 “출동/배정 목록” 버튼을 클릭한다.
2. 시스템은 출동 상세 화면을 출력한다. (A1, A2, A3)
    - 고객 이름, 연락처, 차량 번호
    - 사고 발생 위치 (주소 텍스트)
    - 전화걸기 버튼 (고객 연락처 기반)
    - [현장 도착 처리]버튼
3. 현장출동 직원이 사고 현장에 도착한 뒤 [현장 도착 처리] 버튼을 클릭한다.
4. 시스템은 도착 시간을 저장하고 현장 기록 입력 폼으로 화면을 전환하여 출력한다.
    - 필수 사진 촬영 버튼 영역 (전경 사진, 파손 부위 근접 사진, 차량 번호판 사진, 블랙박스 영상 확보 사진)
    - [추가 사진 촬영] 버튼 (선택 사항)
    - 경찰 출동 여부 라디오 버튼 (예, 아니요)
    - 자력 주행 가능 여부 라디오 버튼 (가능, 불가능-견인 필요)
    - 현장 특이사항 및 요원 소견 입력 텍스트 박스
    - 비활성화된 [기록 전송하기] 버튼
5. 현장출동 직원은 필수 사진 촬영 버튼을 각각 클릭하여 사진을 촬영하고 업로드한다. 경찰 출동 필요와 견인 필요에 체크하고 특이사항을 텍스트 박스에 기입한다. (E1)
6. 시스템은 필수 사진이 모두 업로드되고 필수 항목이 선택된 것을 감지하여 [기록 전송하기] 버튼을 활성화한다.
7. 현장출동 직원이 활성화된 [기록 전송하기] 버튼을 클릭한다.
8. 시스템은 기록된 데이터를 “전송이 완료되었습니다.” 팝업을 띄운 뒤 '출동/배정 목록' 메인 화면으로 되돌아간다.

**Alternative**

A1) 고객이 사고 발생 위치를 업데이트 한 경우

1. 시스템이 화면에 “고객이 위치 정보를 업데이트 했습니다.”라는 알림 문구와 함께 알림 소리를 낸 뒤 Basic Path 2번으로 돌아간다.

---

A2) 현장 도착 전 고객에게 미리 연락을 취하는 경우

1. 현장출동 직원이 고객의 정확한 위치를 파악하기 위해 [전화걸기] 버튼을 클릭한다.
2. 시스템은 직원의 스마트폰 기본 전화 앱을 실행하여 고객의 연락처로 전화를 즉시 연결한다.
3. 직원은 통화를 마치고 [통화 종료] 버튼을 클릭한다.
4. 시스템은 Basic Path 2번으로 돌아간다.

---

**A3) 사고 상황 종료로 출동이 취소되는 경우**

1. 시스템은 알림음과 함께 “고객님께서 출동 요청을 취소하였습니다.”라는 메시지와 함께 취소 사유, [확인] 버튼을 출력한다.
2. 현장출동 직원이 [확인] 버튼을 클릭한다.
3. Basic Path 1번으로 돌아간다.

**Exception**

**E1) 필수 입력 항목 누락**

1. 현장출동 직원이 일부 필수 입력 항목을 누락한 채로 다음 항목들을 기입한다.
2. 시스템은 ‘기록 전송하기’ 버튼을 계속 비활성화(회색) 상태로 유지하며 누락된 항목의 테두리를 붉은색으로 깜빡이게 하여 입력을 유도한다.
3. 직원이 누락된 부분을 찾아 사진을 촬영하거나 버튼을 선택한다.
4. 시스템이 [기록 전송하기] 버튼을 활성화한다.
5. Basic Path 7번으로 이동한다.

## 손해를 조사한다

**Basic Path**

1. 보상담당자가 화면에서 신규 배정건 조회 탭을 클릭한다.
2. 시스템은 배정된 신규 사고/청구 목록 화면을 출력한다.
    - 목록 그리드 (사고 접수 번호, 사고 일자, 고객명, 사고 유형, 처리 상태)
    - 상세 보기 버튼
3. 보상담당자가 특정 접수건의 ‘상세 보기’ 버튼을 클릭한다.
4. 시스템은 해당 사고의 통합 손해조사 상세 화면을 출력한다.
    - **[사고 기본 정보]** 접수 번호, 사고 장소, 고객 연락처
    - **[현장출동 요원 기록]** 현장 사진 썸네일, 블랙박스 영상 링크, 요원 메모 (읽기 전용)
    - **[고객 청구 서류]** 수리 견적서, 영수증 뷰어 버튼
    - **[심사 결과 폼]**
        - 우리 고객 과실 비율 입력 박스 (%)
        - 상대방 과실 비율 입력 박스 (%)
        - 조사 의견 및 합의 내용 텍스트 박스
        - 처리 결과 라디오 버튼 (지급 승인, 면책/지급 거절)
    - 보완 서류 요청 버튼
    - 추가 조사 지시 버튼
    - 비활성화된 ‘조사 완료 및 산출 이관’ 버튼
5. 보상 담당자는 현장 사진과 견적서를 확인한 뒤, 과실 비율을 입력하고 조사 의견 텍스트 박스에 내용을 작성한다. 처리 결과 라디오 버튼에서 ‘지급 승인’을 선택한다. (A1, A2, A3) (E1, E2)
6. 시스템은 필수 입력값이 모두 채워지고 과실 비율의 합을 검증한 뒤, ‘조사 완료 및 산출 이관’ 버튼을 활성화하여 출력한다.
7. 보상 담당자는 활성화된 ‘조사 완료 및 산출 이관’ 버튼을 클릭한다.
8. 시스템은 “조사 내역을 최종 저장하고 보험금 산출 단계로 이동하시겠습니까?”라는 안내 문구와 함께 “확인”, “취소” 버튼이 있는 팝업을 출력한다.
9. 보상 담당자가 “확인” 버튼을 클릭한다.
10. 시스템은 내역 저장 후 상태를 '조사 완료'로 변경하고 보험금 산출 로직을 실행하기 위해 [보험금을 산출한다](https://www.notion.so/341059fb2d6f802a87d4eebdf75be0e9?pvs=21) 유스케이스의 1번으로 이동한다.

**Alternative**

**A1) 제출된 서류가 미비하여 고객에게 보완을 요청하는 경우**

1. 보상담당자가 고객 청구 서류(수리 견적서 등)를 확인했으나, 내용이 흐릿하거나 누락된 항목이 있어 화면의 **‘보완 서류 요청’ 버튼**을 클릭한다.
2. 시스템은 요청 서류 항목 체크박스, 메시지 입력 박스, 발송 버튼, 취소 버튼이 포함된 팝업창을 출력한다.
3. 보상담당자가 필요한 서류 항목을 체크하고 '발송' 버튼을 클릭한다.
4. 시스템은 고객에게 보완 안내 알림(알림톡/문자)을 발송한다. 
5. Basic Path 4번으로 돌아간다.

---

**A2) 내용이 복잡하거나 사기가 의심되어 추가 조사하는 경우**

1. 보상담당자가 현장출동 요원의 기록과 고객에게 전달받은 서류만으로는 과실 판정이 어렵다고 판단하여 화면의 **‘추가 조사 지시’ 버튼**을 클릭한다.
2. 시스템은 방문지(현장, 정비소, 병원) 선택 박스, 일정 입력을 위한 날짜 선택 박스, 추가 조사 사유 입력 박스, 등록 버튼, 취소 버튼을 출력한다.
3. 보상담당자가 방문지를 정비소로 선택하고 일정을 입력한 뒤 등록을 클릭한다.
4. 시스템은 “추가 조사 지시를 완료했습니다.”라는 알림창과 확인 버튼을 출력한다.
5. 보상담당자는 확인 버튼을 클릭한다.
6. Basic Path 4번으로 돌아간다.

---

**A3) 약관상 보상 대상이 아니어서 면책(지급 거절) 처리하는 경우**

1. 보상담당자가 해당 사고가 보상 범위를 벗어난다고 판단하여 처리 결과 라디오 버튼에서 ‘면책/지급 거절’을 선택한다.
2. 시스템은 면책/지급 거절 사유 입력 박스를 출력하고 '조사 완료 및 산출 이관' 버튼을 '조사 완료 및 종결 처리' 버튼으로 변경한다.
3. 보상담당자는 거절 사유를 입력한다.
4. 시스템은 '조사 완료 및 종결 처리' 버튼을 활성화한다.
5. 보상담당자가 버튼을 클릭한다.
6. 시스템은 “해당 사건은 종결 처리됩니다.” 메시지와 확인 버튼, 취소 버튼을 출력한다.
7. 보상담당자는 확인 버튼을 클릭한다.
8. 시스템은 고객에게 면책 사유 안내문을 발송하고 Basic Path 2번으로 돌아간다.

**Exception**

**E1.)과실 비율 입력 오류**

1. 보상담당자가 고객과 상대방의 과실 비율 합이 100이 되지 않도록 입력한다. (예: 60, 50 입력 또는 70, 20 입력)
2. 시스템은 입력값을 검증하여 합산 오류를 감지한다.
3. 시스템은 ‘조사 완료 및 산출 이관’ 버튼을 **비활성화 상태로 유지**하고, 과실 비율 입력 박스 하단에 붉은색 텍스트로 "과실 비율의 합은 반드시 100%이어야 합니다."라는 경고 메시지를 출력한다.
4. 보상담당자가 비율의 합이 100이 되도록 수정한다.
5. 시스템이 다시 검증하여 버튼을 활성화한다.
6. Basic Path 7번으로 이동한다.

---

**E2) 필수 입력값 누락**

1. 보상담당자가 일부 필수 기입 항목을 입력하지 않고 다음 항목을 기입한다.
2. 시스템이 ‘조사 완료 및 산출 이관’ 버튼을 **비활성화 상태로 유지**하고 필수 입력 항목의 테두리를 붉은색으로 강조하여 입력을 유도한다.
3. 보상담당자가 누락된 필수 입력 항목을 입력한다.
4. 시스템이 버튼을 활성화한다.
5. Basic Path 7번으로 이동한다.

## 보험금을 산출한다

**Basic Path**

1. [손해를 조사한다](https://www.notion.so/341059fb2d6f804e9027cd7203e5ee3a?pvs=21) 유스케이스 10번에서 넘어온다.
2. 시스템은 '총 인정 손해액'과 '과실 비율', '보험 가입 약관(보장 한도, 자기부담금 설정율)' 내역을 정해진 보험금 산출 공식에 적용하여 최종 지급액을 자동 산출하여 산출 내역 상세 화면을 출력한다. (E1, E2)
    
    (예) (총 인정 손해액 x 고객 과실 비율) - 자기부담금
    
    - [산출 기초 데이터] 총 인정 손해액, 적용된 고객 과실 비율
    - [공제 내역] 자기부담금(또는 면책금) 적용액
    - [최종 산출액] 최종 지급 산출 금액 텍스트
    - 결재선 지정 선택 박스
    - 지급 승인 및 이관 버튼
    - 이전 페이지로 이동하기 버튼
    - 취소 버튼
3. 보상담당자는 확인 후 ‘지급 승인 및 이관’ 버튼을 클릭한다. (A1, A2)
4. 시스템은 최종 지급액을 확정하고 지급 단계로 이관하시겠습니까?"라는 문구와 함께 "확인", "취소" 버튼이 있는 팝업을 출력한다.
5. 보상 담당자는 사용자가 "확인" 버튼을 클릭한다. (A3)
6. 시스템은 최종 금액을 지급 예정 데이터로 저장하고, 접수건의 상태를 '산출 완료'로 변경한다.
7. [보험금을 지급한다](https://www.notion.so/341059fb2d6f8011ac0df0063f0220ca?pvs=21) 유스케이스의 1번으로 이동한다.

**Alternative**

**A1) 산출액이 커서 상위 결재권자의 승인이 필요한 경우**

1. 기본 흐름 2번 단계에서 계산된 최종 산출액이 보상담당자의 전결 한도를 초과하는 경우.
2. 보상담당자가 화면의 ‘결재선 지정 선택 박스’를 클릭하여 상위 결재권자를 선택한다.
3. 시스템은 ‘지급 승인 및 이관’ 버튼의 텍스트를 ‘결재 상신하기’로 변경하여 출력한다.
4. 보상담당자가 '결재 상신하기' 버튼을 클릭한다.
5. 시스템은 상위 결재권자에게 알림을 발송한다.
6. Basic Path 2번으로 이동한다.

**A2) 산출 화면에서 진행을 멈추고 취소(뒤로가기)하는 경우**

1. 보상담당자가 처리를 일시 보류하거나 다른 업무를 먼저 처리하기 위해 화면 하단의 이전 페이지로 이동하기 **버튼**을 클릭한다.
2. 시스템은 손해조사 상세 화면을 출력한다.

**A3) 팝업에서 이관을 취소하는 경우**

1. 보상담당자가 내역을 다시 검토하기 위해 "취소" 버튼을 클릭한다.
2. 시스템은 팝업을 닫고 신규 사고/청구 목록 화면을 출력한다.

**Exception**

**E1) 공제액(자기부담금) 초과로 실지급액이 0원 이하가 되는 경우**

1. 시스템이 산출 공식을 적용한 결과가 인정 손해액보다 고객의 자기부담금이 더 커서 최종 산출액이 마이너스(-) 혹은 0원이 되면 최종 지급 산출 금액을 '0원'으로 강제 보정하여 출력하고, 화면 상단에 붉은색 텍스트로 "자기부담금 초과로 실지급 보험금이 없습니다."라는 안내 배너를 띄운다. ‘지급 승인 및 이관’ 버튼을 **‘공제액 초과 종결 처리’** 버튼으로 대체하여 출력한다.
2. 보상담당자가 해당 버튼을 클릭한다.
3. 시스템은 고객에게 면책(지급액 없음) 내용이 담긴 안내문을 발송한다.
4. 시스템은 신규 사고/청구 목록 화면으로 이동한다.

2. Basic Path 3번으로 이동한다.

## 보험금을 지급한다

**Basic Path**

1. [보험금을 산출한다](https://www.notion.so/341059fb2d6f802a87d4eebdf75be0e9?pvs=21) 유스케이스 7번에서 넘어온다.
2. 시스템은 최종 지급 승인 화면을 출력한다.
    - [지급 대상 정보] 접수 번호, 피보험자명, 사고 일자
    - [지급 금액 정보] 최종 확정 지급액
    - [수령 계좌 정보] 수령인명, 은행명, 계좌번호 (마스킹 처리된 일부 별표시)
    - 즉시 지급 / 예약 지급 선택 라디오 버튼 (기본값: 즉시 지급)
    - 보안 인증 비밀번호(또는 결재용 OTP) 입력 박스
    - 이체 후 안내 메시지 보내기 체크 박스 (알림톡, 문자)
    - 이전 페이지로 이동하기 버튼
    - 취소 버튼
    - 비활성화된 ‘이체 실행’ 버튼
3. 보상 담당자는 안내 메시지 보내기 체크 박스에서 알림톡을 선택하고 최종 보안 인증 비밀번호 6자리를 입력한다. (A1, A2)
4. 시스템은 비밀번호가 일치함을 확인하고 ‘이체 실행’ 버튼을 활성화하여 출력한다. (E1)
5. 사용자가 ‘이체 실행’ 버튼을 클릭한다.
6. 시스템은 지정된 계좌로 이체를 요청하고 은행으로부터 이체 성공 결과를 출력한다. (E2)
    - “보험금 지급을 완료하였습니다.” 메시지
    - 수령인명, 은행명, 계좌번호, 이체금액
    - 확인 버튼
7. 보상 담당자는 확인 버튼을 클릭한다.
8. 시스템은 "해당 사고건이 종결 처리되었습니다."라는 문구와 "확인" 버튼이 있는 안내 팝업을 출력한다.
9. 사용자는 확인 버튼을 클릭한다.
10. 시스템은 고객의 휴대폰으로 보험금 지급이 완료되었다는 안내 메시지(알림톡/문자)를 발송하고 “메시지를 전송했습니다.”라는 문구와 확인 버튼이 있는 안내 팝업을 출력한다.
11. 사용자는 확인 버튼을 클릭한다.
12. 시스템은 신규 사고/청구 목록 화면을 출력한다.

**Alternative**

**A1) 당장 지급하지 않고 '예약 지급'을 선택하는 경우**

1. 보상담당자가 당일 은행 점검 시간을 고려하여 **'예약 지급' 라디오 버튼**을 선택한다.
2. 시스템은 예약 일시를 지정할 수 있는 날짜/시간 선택 달력을 화면에 추가로 출력한다.
3. 보상담당자가 일시를 지정하고 비밀번호를 입력한다.
4. 시스템은 버튼명을 '이체 실행'에서 '예약 등록'으로 변경하여 활성화한다.
5. 보상담당자가 '예약 등록' 버튼을 클릭한다.
6. 시스템은 "지정된 일시에 이체가 실행됩니다."라는 메시지와 확인 버튼이 담긴 팝업창을 출력한다.
7. 보상담당자는 확인 버튼을 클릭한다.
8. 시스템은 신규 사고/청구 목록 화면을 출력한다.

---

**A2) 진행을 멈추고 취소(뒤로가기)하는 경우**

1. 보상담당자가 지급 내역을 다시 검토하기 위해 **'**이전 페이지로 이동하기’ **버튼**을 클릭한다.
2. 시스템은 이체 지시를 내리지 않고 기존 상태('산출 완료')를 유지한 채 산출 내역 상세 화면을 출력한다.

**Exception**

**E1) 보안 인증 비밀번호 불일치**

1. 시스템은 비밀번호 불일치를 확인한 뒤 '이체 실행' 버튼을 비활성화 상태로 유지하고, 비밀번호 입력 박스 하단에 붉은색 텍스트로 "비밀번호가 일치하지 않습니다."라는 경고 메시지를 출력한다.
2. 보상담당자가 올바른 비밀번호를 다시 입력한다.
3. 시스템이 다시 검증하여 버튼을 활성화한다.
4. Basic Path 5번으로 이동한다.

---

**E2) 수령 계좌 이상으로 인한 은행 이체 거절**

1. 시스템이 은행 망으로부터 **입금 불가(해지된 계좌, 거래 정지, 예금주 불일치 등)** 결과를 반환받고 성공 결과 화면 대신 "이체가 실패했습니다. 고객의 계좌 상태를 확인해 주세요. (사유: 해지된 계좌)"라는 에러 팝업을 출력한다.
2. 보상담당자가 '확인' 버튼을 누른다.
3. 시스템은 이체 결과 출력 페이지의 '확인' 버튼을 “수령 불가 안내 알림톡 보내기” 버튼으로 변경하여 출력한다.
4. 보상담당자는 “안내 알림톡 보내기” 버튼을 클릭한다.
5. 시스템은 고객의 휴대폰 번호로 "[OO보험] 등록하신 계좌로 보험금 입금이 불가합니다. 새로운 수령 계좌를 앱이나 고객센터를 통해 등록해 주세요."라는 계좌 변경 안내 알림톡을 자동 발송하고 “정상적으로 처리되었습니다.”라는 팝업창을 출력한다.
6. 보상담당자는 확인 버튼을 클릭한다.
7. 시스템은 신규 사고/청구 목록 화면을 출력한다.

## 가입 보험을 조회한다

**Basic Path**

**1.** 고객이 앱 메인 화면에서 '보험료 조회/납부' 메뉴를 클릭한다.

**2.** 시스템은 고객 명의로 가입된 전체 보험 목록 화면을 출력한다. (A1)

- 계약 상태 필터 선택 박스 (전체 / 정상 유지 / 실효 / 만기 / 해약) (기본 설정: 전체)
- 정렬 옵션 선택 박스 (계약일 최신순 / 보험료 높은순) (기본 설정: 계약일 최신순)
- 보험 정보 카드
    - 계약된 보험명 및 증권번호 리스트
    - 보험 계약 상태 (정상 유지 / 실효 / 만기 / 해약)
    - 피보험자명
    - 계약자명
    - 월 보험료, 총 납입 횟수
    - 상세 조회 버튼
- 보험료 납입하기 버튼
- 뒤로가기 버튼

**3.** 고객이 조회를 원하는 보험 계약의 '상세 조회' 버튼을 클릭한다. (A2)

**4.** 시스템은 해당 보험의 상세 화면을 출력한다. 기본정보 탭이 기본 선택된 상태로 출력된다. (E1)

- 보험 계약 상태 (정상 유지 / 실효 / 만기 / 해약)
- **[기본정보 탭]** (기본 선택)
    - 기본정보 카드 박스
        - 상품명, 계약 일자, 증권번호, 보험 기간
        - 계약자명, 피보험자명
        - 취소/해약 버튼
    - 납입정보 카드 박스
        - 납입기간, 납입 보험료, 납입횟수
        - 총 납입 보험료, 최종 납입월, 납입방법
        - 납입일자, 은행명, 계좌번호
        - 납입방법 변경 버튼
    - 약관 확인 버튼
    - 가입 증명서 확인 버튼
- **[보장내용 탭]**
- **[납입내역 탭]**
- 목록 버튼

**5.** 고객은 내용 확인 후 보장내용 탭을 클릭한다. (A3, A4, A5, A6)

**6.** 시스템은 보장내용 탭 화면을 출력한다.

- 보험 계약 상태 (정상 유지 / 실효 / 만기 / 해약)
- **[기본정보 탭]**
- **[보장내용 탭]** (선택됨)
    - 가입자명
    - 가입 담보명 및 보장금액 리스트
- **[납입내역 탭]**
- 목록 버튼

**7.** 고객은 내용 확인 후 납입내역 탭을 클릭한다.

**8.** 시스템은 납입내역 탭 화면을 출력한다.

- **[기본정보 탭]**
- **[보장내용 탭]**
- **[납입내역 탭]** (선택됨)
    - 총 납입 금액, 총 납입 횟수
    - 납입 내역 기간 선택 박스 (기본 설정: 최근 1년)
        - 전체
        - 최근 1년
        - 최근 2년
    - 납입 내역 리스트
        - 납입 날짜
        - 납입 보험료
        - 납입 방법 (자동이체 / 카드납 / 가상계좌 등)
- 목록 버튼

**9.** 고객은 내역 확인 후 목록 버튼을 클릭한다. (A7)

**10.** 시스템은 Basic Path 2번으로 이동한다.

**Alternative**

**A1) 가입된 보험 계약 내역이 없는 경우**

1. 시스템은 계약 목록 대신 빈 화면에 "가입된 보험 계약 내역이 없습니다."라는 안내 텍스트, “메인으로 가기” 버튼, 상품 가입을 유도하는 “**추천 상품 보러가기” 버튼**을 출력한다.
2. 고객이 화면 하단의 “메인으로 가기” 버튼을 클릭한다.
3. 시스템은 메인 화면을 출력한다.

---

A2) 고객이 보험료를 납입하려는 경우

1. 고객이 보험료 납입하기 버튼을 클릭한다.
2. [보험료를 납입한다](https://www.notion.so/341059fb2d6f80929579df3b91fef026?pvs=21) 유스케이스 1번으로 이동한다.

A3) 고객이 보험을 해약하려는 경우

1. 고객이 “취소/해약” 버튼을 클릭한다.  
2. [보험을 해지한다](https://www.notion.so/341059fb2d6f80b6bb68d0a728dbeade?pvs=21) 유스케이스 1번으로 이동한다.

**A4) 납입 방법(계좌/카드)을 변경하려는 경우**

1. 고객이 기본정보 탭의 납입정보 카드 박스에서 '납입방법 변경' 버튼을 클릭한다.
2. 시스템은 현재 등록된 결제 수단을 보여주고, 새로운 결제 수단을 등록할 수 있는 납입방법 변경 화면을 출력한다.
    - 납입방법 (계좌이체 [기본 설정], 카드결제)
    - 은행명 (카드결제 선택 시 카드사명)
    - 계좌번호 (카드결제 선택 시 카드번호)
    - (카드결제 선택 시 유효기간)
    - 다음 버튼
3. 고객이 계좌이체 방식을 선택하여 은행명과 계좌번호를 입력하고 다음 버튼을 클릭한다.

5. 고객은 계좌 내역을 확인하여 인증번호를 입력한다.
6. 시스템은 인증을 확인하고 “납입 방법이 정상적으로 변경되었습니다.”라는 메시지와 확인 버튼이 담긴 팝업창을 출력한다.

8. Basic Path 4번으로 이동한다.

## 보험을 해지한다

**Basic Path**

1. [가입 보험을 조회한다](https://www.notion.so/341059fb2d6f80d9bffbf31378ba47ca?pvs=21) 유스케이스의 A3) 2번에서 넘어온다.
2. 시스템은 고객의 해지 사유를 묻는 해지 사유 입력 폼을 출력한다.
    - 해지 사유 선택 드롭다운 (경제적 사정, 타사 가입, 서비스 불만, 보장 내용 불필요, 기타)
    - 상세 사유 입력 텍스트 박스 (기타 항목에 대해서만 필수입력 사항)
    - 비활성화된 다음 버튼
    - 취소 버튼
3. 고객이 드롭다운에서 '경제적 사정'을 선택하고 '다음' 버튼을 클릭한다. (A1, A2)
4. 시스템은 해지 시 발생할 수 있는 불이익을 안내하는 해지 유의사항 확인 화면을 출력한다.
    - 안내 텍스트: "해약 시점부터 사고 보장이 중단되며, 납입한 보험료보다 해약 환급금이 적거나 없을 수 있습니다."
    - '위 유의사항을 모두 읽고 이해했습니다.' 동의 체크 박스
    - 비활성화된 '본인 인증하고 계속하기' 버튼
    - 취소 버튼
5. 고객이 동의 체크 박스를 확인하고 '본인 인증하고 계속하기' 버튼을 클릭하여 인증한다. (A2) 
6. 시스템은 인증 성공 값을 확인하고 해약을 위한 확인 페이지를 출력한다. (E1)
    - [신청 내역] 선택된 보험명, 증권 번호, 해약 사유
    - 예상 환급 보험료
    - 안내 텍스트: "해약이 완료되면 담당자가 내용을 확인한 후, 2~3일 내에 최종 처리 및 환급 보험료 지급을 진행합니다."
    - 해약하기 버튼
    - 취소 버튼
7. 고객이 화면의 내역을 확인한 후 '해약하기' 버튼을 클릭한다. (A2) (E2)
8. 시스템은 "보험 해약이 완료되었습니다. 환급금은 추후 별도 안내 드리겠습니다.”라는 완료 팝업을 출력한다.
9. [가입 보험을 조회한다](https://www.notion.so/341059fb2d6f80d9bffbf31378ba47ca?pvs=21) 유스케이스의 Basic Path 2번으로 이동한다.

**Alternative**

**A1) 고객이 해지 사유로 '기타'를 선택하는 경우**

1. 고객이 해지 사유 드롭다운에서 '기타'를 선택한다.
2. 시스템은 상세 사유 입력 텍스트 박스를 필수 입력 항목으로 표시하며, '다음' 버튼은 비활성화 상태를 유지한다.
3. 고객이 상세 사유 텍스트 박스에 내용을 입력한다.
4. 시스템은 텍스트 박스에 1자 이상 입력된 것을 확인하고 '다음' 버튼을 활성화한다.
5. Basic Path 3번으로 이동한다.

---

**A2) 해약 중간에 취소하는 경우**

1. 고객이 '취소' 버튼을 클릭한다.
2. 시스템은 "해약을 취소하고 보험 상세 화면으로 돌아가시겠습니까?" 확인 팝업을 출력한다. ('확인' 버튼, '닫기' 버튼 포함)
3. 고객이 '확인' 버튼을 클릭한다.

**Exception**

4. 고객은 '다시 시도' 버튼을 클릭한다. 
5. 시스템은 Basic Path 6번으로 돌아간다.

## 보험료를 납입한다

**Basic Path**

1. [가입 보험을 조회한다](https://www.notion.so/341059fb2d6f80d9bffbf31378ba47ca?pvs=21) 유스케이스 A2) 2번에서 넘어온다.
2. 시스템은 납입할 보험을 선택하는 화면을 출력한다. 계약 상태가 '정상 유지'인 보험만 목록에 표시된다. (A1)
    - 안내 문구: "납입 신청할 보험을 선택해 주세요."
    - 보험 목록 (각 항목 왼쪽에 체크박스)
        - 보험명, 증권번호
        - 월 보험료
        - 주피보험자명
        - 최종 납입월
        - 납입방법
    - "총 n건 선택하기" 버튼 (선택된 항목이 1건 이상일 때 활성화, 0건일 때 비활성화)
    - ‘취소’ 버튼

**3.** 고객은 납입하고자 하는 보험을 1건 이상 선택한 뒤 "총 n건 선택하기" 버튼을 클릭한다. (A)

**4.** 시스템은 납입 횟수를 입력하는 화면을 출력한다.

- 안내 문구: "납입 횟수를 입력해 주세요."
- 선택된 보험 목록
    - 보험명, 증권번호
    - 월 보험료
    - 주피보험자명
- 납입 횟수 입력 박스 (입력 가능 범위: 1 이상 ~ 잔여 납입 횟수 이하의 자연수)

- ‘취소’ 버튼

**5.** 고객은 납입 횟수를 입력하고 '다음' 버튼을 클릭한다. (A)

**6.** 시스템은 납입 방법을 선택하는 화면을 출력한다.

- 안내 문구: "납입 방법을 선택해 주세요."
- 납입 방법 선택 (기본 설정: 즉시이체)
    - 즉시이체
    - 가상계좌
- 계좌 선택 (즉시이체 선택 시에만 표시, 기본 설정: 등록된 계좌)
    - 등록된 계좌
    - 새로운 계좌
- 납입 계좌 선택 박스 (즉시이체 선택 후 등록된 계좌 선택 시에만 표시, 등록된 계좌 목록 출력)
- '다음' 버튼
- ‘취소’ 버튼

**7.** 고객은 납입 계좌를 선택 박스에서 계좌를 선택한 뒤 '다음' 버튼을 클릭한다. (A, A2, A3)

**8.** 시스템은 최종 확인 화면을 출력한다.

- 안내 문구: "아래 내용으로 보험료 납입을 신청하시겠어요?"
- 총 신청 금액
    - 실 납입 금액 (선납 할인이 적용된 경우 할인 전 금액과 할인 적용 금액을 함께 표시)
- 보험별 신청 내역
    - 보험명, 증권번호
    - 납입할 보험료 (회당)
    - 납입할 횟수
    - 월 보험료
- 계좌 정보
    - 납입 방법, 이체 계좌 (또는 가상계좌 안내)
- '신청하기' 버튼
- ‘취소’ 버튼

**9.** 고객은 내용 확인 후 '신청하기' 버튼을 클릭한다. (A)

**10.** 시스템은 납입 신청 완료 화면을 출력한다. (E1)

- 안내 문구: "보험료 납입 신청이 완료되었습니다."
- 신청 완료 내역
    - 납입 방법, 이체 계좌
    - 총 납입 금액
    - 신청 일시
- '메인으로 가기' 버튼
- '납입 내역 확인하기' 버튼

**11.** 고객이 '메인으로 가기' 버튼을 클릭한다. (A, A4)

**12.** 시스템은 메인 화면을 출력한다.

## 보험료 납부 내역을 관리한다

**Basic Path**

1. 재무 회계 담당자가 관리자 시스템 메뉴에서 '수납/입금 관리'를 클릭한다.
2. 시스템은 전체 납부 내역 목록 화면을 출력한다.
    - 조회 필터 (기본 설정: 조회 시작일·종료일은 당일 기준 최근 1개월, 수납 상태는 ‘전체’)
        - 조회 시작일, 조회 종료일
        - 수납 상태 (대기 / 완료 / 전체)
    - 조회 버튼
    - 납부 내역 리스트
        - 결제 일자, 결제 번호
        - 고객명, 보험명, 증권번호
        - 결제 금액
        - 결제 수단 (카드 / 계좌이체 / 가상계좌)
        - 수납 상태 (대기 / 완료)
        - 상세 확인 버튼
    - 미납 알림 자동 발송 설정 버튼
    - 뒤로가기 버튼
3. 담당자가 조회 필터의 조회 시작일, 조회 종료일, 수납 상태(대기)를 설정한 뒤 '조회' 버튼을 클릭한다. (A1)
4. 시스템은 필터 조건에 맞는 납부 내역 리스트를 갱신하여 출력한다. (A2)
5. 담당자가 장부에 반영할 특정 납부 레코드의 '상세 확인' 버튼을 클릭한다.
6. 시스템은 해당 납부 건의 수납 상세 정보 화면을 출력한다.
    - 고객명, 보험명, 증권번호
    - 결제 일자, 결제 번호, 결제 수단
    - 실 결제 금액
    - 해당 월 차수 (예: 48회차)
    - 연체료 포함 여부 및 연체료 금액 (연체료가 포함된 경우에만 표시)
    - 결제 승인 번호
    - 연결된 보험 계약의 현재 상태
    - '수납 확정 및 장부 반영' 버튼
    - '수납 반려' 버튼
    - '목록으로 돌아가기' 버튼
7. 담당자가 내용을 확인한 후 '수납 확정 및 장부 반영' 버튼을 클릭한다. (A3, A4)
8. 시스템은 "해당 납부 내역을 수납 원장에 반영하시겠습니까? 완료 후에는 계약 상태가 자동으로 업데이트됩니다."라는 확인 팝업을 출력한다. ('확인' 버튼, '취소' 버튼 포함)
9. 담당자가 팝업에서 '확인' 버튼을 클릭한다. (A5)
10. 시스템은 수납 원장 반영 및 계약 상태 업데이트가 완료되었음을 알리는 "수납 확정이 완료되었습니다. 계약 상태가 업데이트되었습니다." 완료 메시지를 출력하고 해당 건의 수납 상태를 '완료'로 갱신한다. (E1)
11. Basic Path 4번으로 이동한다.

**Alternative**

**A1) 담당자가 미납 알림 자동 발송을 설정하는 경우**

1. 담당자가 목록 화면에서 '미납 알림 자동 발송 설정' 버튼을 클릭한다.
2. 시스템은 미납 알림 자동 발송 설정 화면을 출력한다.
    - 자동 발송 활성화 여부 토글 (기본 설정: 비활성화)
    - 발송 기준일 설정 (납입일 기준 경과 일수 입력 박스, 예: 납입일 경과 후 3일)
    - 발송 메시지 템플릿 미리보기
3. 담당자가 자동 발송 토글을 활성화하고 발송 기준일을 입력한 뒤 '저장' 버튼을 클릭한다.
4. 시스템은 "미납 알림 자동 발송 설정이 저장되었습니다." 완료 메시지를 출력한다.
5. Basic Path 2번의 목록 화면으로 이동한다.

---

A2) 조회 조건에 해당하는 납부 내역이 없는 경우

1. 시스템은 납부 내역 리스트 영역에 "조회 조건에 해당하는 납부 내역이 없습니다." 안내 문구를 출력한다.
2. 담당자가 조회 필터 조건을 변경한 뒤 '조회' 버튼을 클릭한다.
3. Basic Path 4번으로 이동한다.

---

**A3) 담당자가 납부 건을 반려하는 경우**

1. 담당자가 수납 상세 정보 화면에서 '수납 반려' 버튼을 클릭한다.
2. 시스템은 반려 사유를 입력하는 팝업을 출력한다.
    - 반려 사유 선택 드롭다운 (오류 결제 / 이중 납부 / 계약 불일치 / 기타)
    - 상세 사유 입력 텍스트 박스 ('기타' 선택 시 필수 입력)
    - '반려 확정' 버튼, '닫기' 버튼
3. 담당자가 반려 사유를 선택하고 '반려 확정' 버튼을 클릭한다.
4. 시스템은 해당 납부 건의 수납 상태를 '반려'로 갱신하고 "수납 반려 처리가 완료되었습니다." 완료 메시지를 출력한다.
5. Basic Path 4번으로 이동한다.

---

**A4) 담당자가 '목록으로 돌아가기' 버튼을 클릭하는 경우**

1. 담당자가 수납 상세 정보 화면에서 '목록으로 돌아가기' 버튼을 클릭한다.
2. Basic Path 4번으로 이동한다.

---

**A5) 담당자가 수납 확정 팝업에서 취소하는 경우**

1. 담당자가 확인 팝업에서 '취소' 버튼을 클릭한다.
2. Basic Path 6번으로 이동한다.

**Exception**

**E1) 수납 확정 및 장부 반영 처리 중 오류가 발생하는 경우**

1. 처리 중 오류가 발생하여 시스템이 "수납 확정 처리 중 오류가 발생하였습니다. 잠시 후 다시 시도해 주세요." 안내 메시지와 함께 '다시 시도' 버튼과 '목록으로 돌아가기' 버튼을 출력한다.
2. 담당자는 '다시 시도' 버튼을 클릭한다.
3. Basic Path 6번으로 이동한다.

## 해약 환급 내역을 조회한다

**Basic Path**

1. 재무회계 담당자가 관리자 메뉴에서 '환급금 산출내역 조회' 탭을 클릭한다.
2. 시스템은 해지 승인된 환급금 대상 전체 목록 화면을 출력한다.
    - 조회 필터 (기본 설정: 진행 상태 '전체', 고객명·증권번호 미입력)
        - 진행 상태 선택 드롭다운 (전체 / 산출 대기 / 산출 완료 / 지급 완료)
        - 고객명 입력 박스
        - 증권번호 입력 박스
    - 조회 버튼
    - 목록 그리드
        - 접수 번호, 진행 상태, 고객명, 증권번호, 해지 승인일
3. 재무회계 담당자가 조회 필터에 원하는 조건을 입력하고 '조회' 버튼을 클릭한다.
4. 시스템은 조회 조건에 맞는 목록 그리드를 갱신하여 출력한다. (A1)
    - 접수 번호, 진행 상태, 고객명, 증권번호, 해지 승인일
5. 재무회계  담당자가 상세 내용을 확인할 계약 행을 클릭한다.
6. 시스템은 해당 건의 환급금 산출내역 상세 화면을 출력한다. (E1)
    - [상태 정보] 현재 진행 상태 (산출 대기 / 산출 완료 / 지급 완료)
    - [계약 정보] 고객명, 증권번호, 해지 사유, 수령 계좌 정보 (은행명, 계좌번호, 예금주명)
    - [산출 내역] 기본 환급금, 공제액, 최종 실지급액
        - '산출 대기' 상태인 경우 금액란은 '-'으로 표시
    - [버튼 영역] 진행 상태에 따라 아래와 같이 표시
        - '산출 대기': '환급금 산출하기' 버튼 활성화, '환급금 지급하기' 버튼 비활성화
        - '산출 완료': '환급금 산출하기' 버튼 비활성화, '환급금 지급하기' 버튼 활성화
        - '지급 완료': '환급금 산출하기' 버튼 비활성화, '환급금 지급하기' 버튼 비활성화
    - 목록으로 돌아가기 버튼
7. 재무회계  담당자는 내용을 확인한 후 '목록으로 돌아가기' 버튼을 클릭한다. (A2, A3)
8. 시스템은 Basic Path 2번의 목록 화면을 출력한다.

**Alternative**

**A1) 조회 조건에 해당하는 내역이 없는 경우**

1. 시스템은 목록 그리드 영역에 "조회 조건에 해당하는 환급금 내역이 없습니다." 안내 문구를 출력한다.
2. 재무회계  담당자가 조회 필터 조건을 변경한 뒤 '조회' 버튼을 클릭한다.
3. Basic Path 4번으로 이동한다.

---

**A2) 산출 대기 건을 조회한 후 환급금을 산출하는 경우**

1. 재무회계 담당자가 산출 내역의 금액란이 '-'로 표시된 것을 확인하고 활성화된 '환급금 산출하기' 버튼을 클릭한다.
2. [해약 환급금을 산출한다](https://www.notion.so/341059fb2d6f80b5b982cb6dc24910f0?pvs=21) 유스케이스 1번으로 이동한다.
3. [해약 환급금을 산출한다](https://www.notion.so/341059fb2d6f80b5b982cb6dc24910f0?pvs=21) 유스케이스가 완료되어 복귀한다.
4. 시스템은 진행 상태를 '산출 완료'로 갱신하고, 산출된 기본 환급금·공제액·최종 실지급액을 반영하여 상세 화면을 갱신 출력한다.
5. Basic Path 6번으로 이동한다.

---

**A3) 산출 완료 건을 조회한 후 환급금을 지급하는 경우**

1. 재무회계  담당자가 산출 내역의 최종 실지급액을 확인하고 활성화된 '환급금 지급하기' 버튼을 클릭한다.
2. [환급금을 지급한다](https://www.notion.so/341059fb2d6f802abcecfd16d4d30104?pvs=21) 유스케이스 1번으로 이동한다.
3. [환급금을 지급한다](https://www.notion.so/341059fb2d6f802abcecfd16d4d30104?pvs=21) 유스케이스가 완료되어 복귀한다.
4. 시스템은 진행 상태를 '지급 완료'로 갱신하고, '환급금 지급하기' 버튼을 비활성화하여 상세 화면을 갱신 출력한다.
5. Basic Path 6번으로 이동한다.

**Exception**

**E1) 상세 정보를 조회할 수 없는 경우**

1. 시스템 이상이 발생하여 상세 화면 대신 "해당 건의 상세 정보를 현재 조회할 수 없습니다. 잠시 후 다시 시도하거나 담당 부서에 문의해 주세요." 안내 메시지와 함께 '다시 시도' 버튼과 '목록으로 돌아가기' 버튼을 출력한다.
2. 재무회계 담당자가 '다시 시도' 버튼을 클릭한다. 
3. Basic Path 6번으로 이동한다.

## 해약 환급금을 산출한다

**Basic Path**

1. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스 A2) 2번에서 넘어온다.
2. 시스템은 해당 건의 계약 데이터(총 납입 보험료, 납입 기간, 책임준비금, 적용 이율, 미납 보험료, 약관 대출 원금·이자)를 기반으로 기본 해약 환급금 및 공제 내역을 산출한다.
3. 시스템은 환급금 산출 내역 검증 화면을 출력한다.
    - [기본 정보] 고객명, 증권번호, 해지 승인일
    - [산출 기초] 총 납입 보험료, 책임준비금, 기본 해약 환급금
    - [공제 내역] 미납 보험료, 약관 대출 원금, 약관 대출 이자
    - [최종 금액] 실지급 해약 환급금 (기본 해약 환급금 - 공제 내역 합계)
    - 산출 내역서 다운로드(PDF) 버튼
    - '환급금 확정 및 지급 이관' 버튼
    - '목록으로 돌아가기' 버튼
4. 재무회계 담당자가 이상이 없음을 확인하고 '환급금 확정 및 지급 이관' 버튼을 클릭한다. (A1, A2, A3)
5. 시스템은 "최종 환급금을 확정하고 지급 부서로 이관하시겠습니까?" 확인 팝업을 출력한다. ('확인' 버튼, '취소' 버튼 포함)
6. 재무회계  담당자가 팝업의 '확인' 버튼을 클릭한다. (A4)
7. 시스템은 최종 실지급 해약 환급금을 저장하고 해당 건의 진행 상태를 '산출 완료'로 변경한 뒤 "환급금 산출 및 지급 이관이 완료되었습니다." 완료 메시지를 출력한다. (E2)
8. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스 A1) 3번으로 돌아간다.

**Alternative**

**A1) 산출된 공제 내역에 수기 조정이 필요한 경우**

1. 재무회계  담당자가 산출 내역 검증 화면에서 공제 내역의 금액이 실제와 다름을 확인하고 해당 항목의 수정 아이콘을 클릭한다.
2. 시스템은 해당 공제 항목의 금액을 직접 입력할 수 있는 입력 박스를 활성화하여 출력한다.
3. 재무회계  담당자가 수정할 금액을 입력하고 '적용' 버튼을 클릭한다.
4. 시스템은 변경된 공제 금액을 반영하여 최종 실지급 해약 환급금을 재산출하고, 해당 항목 옆에 "수기 조정됨" 표시를 출력한다.
5. Basic Path 4번으로 이동한다.

---

**A2) 담당자가 산출 내역서를 다운로드하는 경우**

1. 재무회계 담당자가 산출 내역 검증 화면에서 '산출 내역서 다운로드(PDF)' 버튼을 클릭한다.
2. 시스템은 고객명, 증권번호, 산출 기초, 공제 내역, 최종 실지급액이 포함된 산출 내역서를 PDF 파일로 생성하여 다운로드한다.
3. Basic Path 4번으로 이동한다.

---

**A3) 담당자가 목록으로 돌아가려는 경우**

1. 재무회계  담당자가 '목록으로 돌아가기' 버튼을 클릭한다.
2. 시스템은 "산출 내역이 확정되지 않았습니다. 목록으로 돌아가시겠습니까?" 확인 팝업을 출력한다. ('확인' 버튼, '닫기' 버튼 포함)
3. 재무회계  담당자가 '확인' 버튼을 클릭한다.
4. '해약 환급 내역을 조회한다' 유스케이스의 Basic Path 2번 목록 화면으로 이동한다.

---

**A4) 담당자가 확정 팝업에서 취소하는 경우**

1. 담당자가 확인 팝업에서 '취소' 버튼을 클릭한다.
2. 시스템은 팝업을 닫고 Basic Path 3번의 환급금 산출 내역 검증 화면을 그대로 출력한다.
3. Basic Path 4번으로 이동한다.

**Exception**

**E1) 환급금 산출에 필요한 데이터가 누락된 경우**

1. 누락된 값이 존재하여 시스템은 산출 내역 검증 화면 대신 "환급금 산출에 필요한 일부 데이터가 누락되었습니다. 계약 데이터를 확인한 후 다시 시도하거나 담당 부서에 문의해 주세요." 안내 메시지와 함께 누락된 항목명 목록, '목록으로 돌아가기' 버튼을 출력한다.
2. 재무회계 담당자가 '목록으로 돌아가기' 버튼을 클릭한다.
3. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스의 Basic Path 4번으로 이동한다.

---

**E2) 환급금 확정 저장 처리 중 오류가 발생하는 경우**

1. 오류가 발생하여 시스템이 "환급금 확정 처리 중 오류가 발생하였습니다. 잠시 후 다시 시도하거나 담당 부서에 문의해 주세요." 안내 메시지와 함께 '다시 시도' 버튼 및 '목록으로 돌아가기' 버튼을 출력한다. 해당 건의 진행 상태는 '산출 대기'로 유지된다.
2. 재무회계  담당자는 '다시 시도' 버튼을 클릭한다.
3. Basic Path 3번으로 이동한다.

## 환급금을 지급한다

**Basic Path**

1. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스의 A2) 2번에서 넘어온다.
2. 시스템은 확정된 환급금과 고객 계좌 정보를 결합하여 환급금 지급 승인 화면을 출력한다.
    - [지급 대상] 고객명, 해지 보험명, 증권번호
    - [지급 금액] 최종 확정 해약 환급금
    - [수령 계좌] 예금주명, 은행명, 계좌번호 (뒷 4자리 마스킹)
    - 결재용 OTP 번호 입력 박스 (6자리)
    - '이체 실행' 버튼 (비활성화 상태)
    - '목록으로 돌아가기' 버튼
3. 재무회계 담당자가 수령 계좌와 지급 금액을 최종 검토한 후 결재용 OTP 번호 6자리를 입력한다.
4. 시스템은 입력된 OTP 번호를 검증하고 '이체 실행' 버튼을 활성화한다. (E1)
5. 담당자가 활성화된 '이체 실행' 버튼을 클릭한다. (A1)
6. 시스템은 이체를 실행한 뒤 고객 휴대폰으로 "[OO보험] 고객님의 계좌로 해약 환급금이 입금되었습니다. 그동안 이용해 주셔서 감사합니다." 알림톡을 발송한다. 담당자에게 지급 완료 화면을 출력한다. (E2, E3)
    - [지급 결과] "환급금 지급이 정상적으로 완료되었습니다."
    - [지급 내역] 고객명, 증권번호, 지급 금액, 이체 계좌, 이체 완료 일시
    - '확인' 버튼
7. 담당자는 확인 버튼을 클릭한다.
8. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스의 A2) 3번으로 돌아간다.

**Alternative**

**A1) 담당자가 이체 실행 전 지급을 취소하려는 경우**

1. 담당자가 지급 승인 화면에서 '목록으로 돌아가기' 버튼을 클릭한다.
2. 시스템은 "이체가 실행되지 않았습니다. 지급을 취소하고 목록으로 돌아가시겠습니까?" 확인 팝업을 출력한다. ('확인' 버튼, '닫기' 버튼 포함)
3. 담당자가 '확인' 버튼을 클릭한다.
4. [해약 환급 내역을 조회한다](https://www.notion.so/346059fb2d6f80a39acae5167d8320ac?pvs=21) 유스케이스의 Basic Path 4번으로 이동한다.

**Exception**

**E1) OTP 인증에 실패하는 경우**

1. 시스템은 입력된 OTP 번호가 유효하지 않음을 감지하고 "OTP 번호가 일치하지 않습니다. 다시 입력해 주세요. (남은 시도 횟수: n회)" 안내 메시지를 입력 박스 하단에 출력한다. '이체 실행' 버튼은 비활성화 상태를 유지한다.
2. 담당자가 OTP 번호를 다시 입력한다.
3. 인증 성공 시 Basic Path 4번으로 이동한다.
4. OTP 인증을 5회 연속 실패한 경우, 시스템은 입력 박스와 '이체 실행' 버튼을 비활성화하고 "OTP 인증 시도 횟수를 초과하였습니다. 보안 담당자에게 문의해 주세요." 안내 메시지를 출력한다.

---

**E2) 이체 처리 중 오류가 발생하는 경우**

1. 이체 처리 중 오류가 발행하여 시스템이 "이체 처리 중 오류가 발생하였습니다. 이체가 실행되지 않았습니다. 잠시 후 다시 시도하거나 담당 부서에 문의해 주세요." 안내 메시지와 함께 '다시 시도' 버튼과 '목록으로 돌아가기' 버튼을 출력한다. 해당 건의 진행 상태는 '산출 완료'로 유지된다.
2. 담당자가 '다시 시도' 버튼을 클릭한다.
3. Basic Path 4번으로 이동한다.

---

**E3) 알림톡 발송에 실패하는 경우**

1. 알림톡 발송에 실패하여 시스템이 지급 완료 화면 내에 "환급금 지급은 완료되었으나, 고객 알림톡 발송에 실패하였습니다. 고객에게 별도로 안내해 주세요." 안내 메시지를 추가로 출력한다.
2. Basic Path 8번으로 이동한다.

## 계약 정보를 조회한다

**Basic Path**

1. 계약관리 담당자는 메인 페이지 상단 메뉴바에서 [계약] 메뉴 하위의 [계약관리] 항목을 클릭한다.
2. 시스템은 화면에 [계약 관리] 페이지를 출력한다. (E1)
    - 화면 상단에는 검색 필터 영역이 표시된다. 필터 항목은 고객번호, 이름, 주민등록번호, 연락처, 계약번호, 보험 종류, 계약일자이며, 각 항목은 텍스트 입력 필드로 구성된다. 보험 종류는 드롭다운으로 제공된다.
    - 검색 필터 영역 하단에는 계약 목록 테이블이 표시된다. 테이블 컬럼은 [계약번호 / 계약자명 / 보험 종류 / 계약기간 / 보험료 / 납입 현황 / 계약 상태]로 구성된다.
    - 초기 진입 시 테이블에는 최근 등록순으로 전체 계약 목록이 페이지 당 20건 출력된다.
    - 만기 임박 계약(만료일 기준 30일 이내)은 계약 상태 컬럼에 "만기 임박" 배지로 강조 표시된다.
3. 계약관리 담당자는 검색 필터 영역에 조회할 계약자의 식별 정보를 입력한다. **(A1)**
    - 입력 가능한 항목: 고객번호(숫자), 이름(텍스트), 주민등록번호(000000-0000000 형식), 연락처(000-0000-0000 형식), 계약번호(숫자), 보험 종류(드롭다운 선택), 계약일자(YY-MM-DD 형식)
    - 하나 이상의 항목을 입력 후 [조회] 버튼을 클릭한다.
4. 시스템은 필터링한 조건에 맞는 계약 목록 테이블을 출력한다. **(A2)**
5. 계약관리 담당자는 계약 목록 테이블에서 해당 계약의 행을 클릭한다.
6. 시스템은 해당 계약의 상세 정보를 새 패널에 출력한다. 출력되는 정보는 다음과 같다:**(A3, A4)**
    - 계약자 기본 정보: 이름, 연락처
    - 계약 정보: 계약 번호, 보험 종류, 계약 기간(시작 연월~종료 연), 월 보험료, 계약 상태(정상/만기임박/실효/해지)
    - 납입 현황: 전체 납입 횟수 대비 정상 납입 횟수, 최근 납입일, 연체 여부 및 연체 횟수
    - 특약 정보: 가입된 특약명, 특약별 보장 내용, 특약 보험료 목록**(A5)**
    - 패널 좌측 상단에는 [편집], [저장] 버튼이 있다.
    - 패널 우측 상단에는 [만기 계약 관리], [계약 통계] 버튼이 있다.

**Alternative**

**A1) 필터링 없이 [조회] 버튼을 누를 경우,**

1. 시스템은 전체 계약 목록을 최근 등록순으로 페이지 당 20건 출력한다.

---

**A2) 필터링과 일치하는 계약이 존재하지 않는 경우,** 

1. 시스템은 “해당 계약 정보를 찾을 수 없습니다. 입력하신 조건을 다시 확인해주세요.”라는 메시지를 출력한다.
    - 검색 필터는 초기화되지 않으며, 입력값이 유지된 상태로 표시된다.
    - 계약 관리 담당자는 필터 조건을 수정한 후 [조회]버튼을 다시 클릭할 수 있다.

---

**A3) 만료일 기준 30일 이내인 계약이 확인된 경우,**

1. 시스템은 계약 상세정보 패널 상단에 “이 계약은 [20XX.XX.XX]에 만료됩니다. 만기까지 [D-XX]남았습니다.”라는 메시지를 출력한다.
2. 계약 관리 담당자는 [만기 계약 관리] 버튼을 클릭한다.
3. [만기 계약을 관리한다](https://www.notion.so/341059fb2d6f804d846ad9d563b31968?pvs=21) 유스케이스 1번으로 이동한다.

---

**A4) 계약 통계 분석이 필요한 경우,**

1. 계약 관리 담당자는 [계약 통계] 버튼을 클릭한다.
2. [계약 통계 정보를 관리한다](https://www.notion.so/341059fb2d6f80319364caf452ed7ca7?pvs=21) 유스케이스 1번으로 이동한다.

---

**A5) 특약 미가입시,** 

1. 시스템은 특약 정보란에 “가입된 특약 없음”을 출력한다.

**Exception**

**E1) 페이지 출력 중 오류가 발생한 경우**

1. 시스템은 “데이터를 불러오는 중에 오류가 발생하였습니다. 잠시 후 다시 시도하거나 담당 부서에 문의해 주세요." 안내 메시지와 함께 '다시 시도' 버튼과 '목록으로 돌아가기' 버튼을 출력한다.
2. 계약 관리 담당자가 '다시 시도' 버튼을 클릭한다. 
3. Basic Path 2번으로 이동한다.

## 계약 통계 정보를 관리한다

**Basic Path**

1. [계약 정보를 조회한다](https://www.notion.so/341059fb2d6f80eaba20ddd0f3532679?pvs=21) A3) 2번에서 넘어온다.
2. 시스템은 계약 통계 페이지를 표시한다. 페이지 구성은 다음과 같다.
    - 페이지 상단 좌측에는 조회 중인 계약 번호와 계약자명이 표시된다.
    - 페이지 상단 우측에는 [전체 계약 통계] 버튼과 [엑셀 다운로드] 버튼이 있다. **(A1)**
    - 보험료 납부 현황 테이블: 납부 회차, 납부일, 납부 금액, 납부 상태(정상/연체/미납)이 나열되어 있고, 전체 납부 이력이 최신순으로 표시된다. 테이블 우측 상단에는 시작 연월과 종료 연월을 입력할 수 있는 기간 필터와 [조회] 버튼이 있다.
    - 보험금 청구 이력 테이블: 청구 일시, 청구 금액, 지급 금액, 처리 상태(지급 완료/심사중/반려)로 구성되어 있고, 전체 청구 이력이 최신 순으로 표시된다.
    - 계약 유지율 그래프: 계약 시작일부터 현재까지의 월별 유지 여부를 막대 그래프로 표시한다.
3. 계약관리 담당자는 보험료 납부 현황 테이블의 기간 필터에 시작 연월과 종료 연월을 입력하고 [조회] 버튼을 클릭한다. **(E1)**
4. 시스템은 입력한 기간에 해당하는 납부 이력만 필터링하여 테이블을 갱신한다.
5. 계약관리 담당자는 [엑셀 다운로드] 버튼을 클릭한다.
6. 시스템은 현재 화면에 표시된 통계 데이터를 엑셀 파일로 생성하여 다운로드한다. 파일명은 “계약통계_[계약번호]_[다운로드일자].xlsx”로 저장된다.

**Alternative**

**A1) 전체 계약 통계를 조회하는 경우**

1. 계약관리 담당자는 [전체 계약 통계] 버튼을 클릭한다.
2. 시스템은 전체 계약 통계 페이지를 표시한다. 페이지 상단에는 필터 조건 영역이 있으며 페이지 하단에는 [조회] 버튼이 있다. 필터 항목은 다음과 같다.
    - 보험 종류: 전체(기본값), 생명보험, 건강보험, 자동차보험, 화재보험
    - 계약 상태: 전체(기본값), 정상, 만기 임박, 실효, 해지
    - 계약 기간: 시작 연월~종료 연월 입력란
3. 계약 관리 담당자는 원하는 필터 조건을 설정하고 [조회] 버튼을 클릭한다. (A2)(E1)
4. 시스템은 필터 조건에 해당하는 계약을 조회하고 페이지 하단에 다음 항목들을 표시한다.
    - 조회 결과 요약: 전체 계약 건수, 정상 건수, 만기 임박 건수, 실효 건수, 해지 건수
    - 월별 신규 계약 건수 추이: 조회 기간 내 월별 신규 계약 건수를 꺾은선 그래프로 표시
    - 보험 종류별 계약 비율: 보험 종류별 계약 건수를 원형 그래프로 표시
    - 보험료 수납 현황 요약: 조회 기간 내 전체 납부 금액, 연체 금액, 미납 금액을 수평 막대 그래프로 표시
    - 조건에 해당하는 계약 목록 테이블: 계약 번호, 계약자명, 보험 종류, 계약 기간, 계약 상태
5. 계약관리 담당자는 계약 목록 테이블에서 특정 계약 행을 클릭한다.
6. basic path 2로 돌아간다.

---

**A2) 필터 조건에 해당하는 계약이 존재하지 않는 경우**

1. 시스템은 조회 결과 영역에 “해당 조건에 일치하는 계약이 존재하지 않습니다.”라는 메시지를 표시한다.
2. 계약관리 담당자는 필터 조건을 수정한 후 [조회]버튼을 클릭한다.

**Exception**

**E1) 기간 필터의 종료 연월이 시작 연월보다 앞선 경우**

1. 계약관리 담당자는 기간 필터에서 종료 연월을 시작 연월보다 이전으로 입력한다.
2. 계약관리 담당자는 [조회] 버튼을 클릭한다.
3. 시스템은 조회를 처리하지 않고, 기간 입력란 하단에 “종료 연월은 시작 연월 이후이어야 합니다.”라는 문구를 빨간색 텍스트로 표시한다.
4. 계약관리 담당자는 기간을 수정한 후 [조회] 버튼을 클릭한다.

## 만기 계약을 관리한다

**Basic Path**

1. [계약 정보를 조회한다](https://www.notion.so/341059fb2d6f80eaba20ddd0f3532679?pvs=21) A3) 2번에서 넘어온다.
2. 시스템은 만기 계약 관리 화면으로 전환한다.
    - 화면 상단에는 대상 계약 정보(계약번호, 계약자명, 보험종류, 만료일, 잔여 일수)가 고정 표시된다.
    - 대상 계약 정보 아래에는 [만기 안내], [갱신 처리], [처리 이력] 탭이 위치한다.
    - 화면 하단에는 [해지 처리로 전환] 버튼이 표시된다.
    - 초기 진입 시 [만기 안내] 탭이 활성화된다.
3. 계약관리 담당자는 [만기 안내] 탭을 클릭한다.
4. 시스템은 [만기 안내] 탭 화면을 출력한다. **(E1)**
    - 화면에는 고객 연락처(계약자명, 전화번호,이메일), 안내 내용 미리보기(만료일, 갱신 가능 여부, 갱신 시 예상 보험료, 고객 응답 링크), [안내 문자 발송] 버튼, [안내 기록] 버튼이 있다.
5. 계약관리 담당자는 [안내 문자 발송] 버튼을 클릭한다.
6. 시스템은 고객의 등록된 전화번호로 만기 안내 문자를 발송한다.
    - 문자에는 만료일, 갱신 가능여부, 갱신 시 예상 보험료, 고객이 갱신/해지/추후 결정 중 하나를 선택할 수 있는 응답 링크가 포함된다. **(E2)**
7. 계약관리 담당자는 [안내 기록] 버튼을 클릭한다.
8. 시스템은 안내 기록 입력 폼을 출력한다.
    - 입력 폼은 안내 일시(자동 입력), 안내 담당자(로그인 계정으로 자동 입력), 메모(텍스트 입력)으로 구성된다.
9. 계약관리 담당자는 메모를 입력한 후 [저장] 버튼을 클릭한다.
10. 시스템은 안내 기록을 저장하고 [처리 이력]탭에 반영한다.
11. 고객은 응답 링크를 통해 의사를 선택한다.
12. 시스템은 계약관리 담당자에게 “고객[고객 이름]님이 [갱신 희망/해지 희망/추후 결정]을 선택하였습니다.” 알림을 발송한다. **(A1, A2, A3)**

**Alternative**

**A1) 고객이 갱신을 희망하는 경우**

1. 계약관리 담당자는 알림을 확인하고 만기 계약 관리 화면 상단의 [갱신 처리] 탭을 클릭한다.
2. 시스템은 “ [고객이름]님의 계약 [계약번호]을 갱신 처리할까요?”라는 팝업 메시지를 출력한다. 팝업 하단에는 [예], [아니오] 버튼이 있다.
3. 계약관리 담당자는 [예] 버튼을 클릭한다.**(A4)**
4. 시스템은 [갱신 처리] 화면을 출력한다. 화면에는 다음 항목이 표시된다:
    - 현재 계약 정보(보험 종류, 계약기간, 현재 월 보험료, 특약 목록)
    - 갱신 후 계약 정보 입력 영역(갱신 계약 기간 - 시작일(자동 입력), 종료일(선택), 갱신 후 월 보험료, 변동액(현재 월 보험료 대비 증감액)
    - [갱신 확정] 버튼
5. 계약관리 담당자는 [갱신 확정] 버튼을 클릭한다.
6. 시스템은 “[고객 이름]님의 계약 [계약번호]이 정상적으로 갱신처리되었습니다.”라는 팝업 메시지를 출력한다.
7. 시스템은 갱신된 계약 정보를 저장하고 [처리 이력] 탭에 반영한다.

---

**A2) 고객이 해지를 희망하는 경우**

1. 계약관리 담당자는 알림을 확인하고 만기 계약 관리 화면 하단의 [해지 처리로 전환] 버튼을 클릭한다.
2. 시스템은 “해지 처리로 전환하시겠습니까?”라는 팝업 메시지를 출력한다. 팝업 하단에는 [예], [아니오] 버튼이 있다.
3. 계약관리 담당자는 [예] 버튼을 클릭한다.**(A4)**

---

**A3) 고객이 추후 결정을 원하는 경우**

1. 계약관리 담당자는 알림을 확인하고 [처리 이력] 탭에 추후 결정 상태로 기록된 것을 확인한다.
2. 만료일 7일 전, 시스템은 계약관리 담당자에게 “결정 대기 중인 만기 계약이 있습니다: [계약번호], 만료까지 D-7”알림을 자동 발송한다.
3. 계약관리 담당자는 알림을 클릭한다.
4. Basic Path 2번으로 돌아간다.

---

**A4) 고객이 [아니오] 버튼을 클릭하는 경우,**

1. 시스템은 팝업을 닫고 만기 계약 관리 화면으로 돌아간다.

**Exception**

**E1) 만료일이 이미 경과한 계약인 경우**

1. 시스템은 만기 계약 관리 화면 상단에 “이 계약은 [YYYY.MM.DD]에 이미 만료되었습니다. 갱신처리는 만료일 이전에만 가능합니다.”라는 배너를 출력한다.
2. 시스템은 [갱신처리]탭을 비활성화 상태로 표시된다.
3. [해지 처리로 전환] 버튼은 활성화 상태를 유지한다.

**E2) 문자 발송에 실패한 경우**

1. 시스템은 “문자 발송에 실패했습니다. 고객 연락처를 확인하거나 직접 안내해주세요.”라는 메시지를 출력한다.
2. 계약관리 담당자는 고객에게 직접 전화하여 만기 안내를 진행한다.
3. basic path 8로 돌아간다.

## 고객 정보를 등록한다

**Basic Path**

1. 판매채널은 메인 페이지 상단 메뉴바에서 [영업 활동] 메뉴 하위의 [고객 등록] 항목을 클릭한다.
2. 시스템은 고객정보 등록 화면을 출력한다.
    - 화면은 [기본 정보], [계약 정보] 섹션으로 구성된다.
    - 화면 우측 상단에 [저장], [편집] 버튼이 배치된다.
    - 고객번호 필드는 “저장 시 자동부여” 안내 문구와 함께 비활성화 상태로 표시된다.
3. 판매채널은 [편집] 버튼을 클릭한다.
4. 판매채널은 [기본 정보] 섹션에 고객의 식별 정보(이름, 주민등록번호, 연락처, 주소)를 입력한다.
    - [기본 정보] 섹션에는 이름, 주민등록번호, 연락처, 주소 필드가 배치된다.
    - 입력 항목(형식), 필수 여부
        - 이름(텍스트), 필수
        - 주민등록번호(000000-0000000, 뒷자리 마스킹), 필수 **(A1)**
        - 연락처(000-0000-0000), 필수
        - 주소(우편번호 검색 후 상세주소 입력), 선택
5. 판매채널은 [계약 정보] 섹션에 계약 관련 정보(보험종류, 계약일, 만료일, 월 보험료, 특약 정보)를 입력한다.
    - [계약 정보] 섹션에는 계약번호, 보험종류, 계약일, 만료일, 월보험료, 특약 정보 필드가 배치된다.
    - 입력 항목(형식), 필수 여부
        - 보험종류(생명/건강/자동차/화재), 드롭다운 선택, 필수
        - 계약일(YYYY.MM.DD), 달력 팝업, 필수
        - 만료일(YYYY.MM.DD), 달력 팝업, 필수
        - 월 보험료(원 단위), 필수
        - 특약 정보(텍스트 입력), 선택**(A2)**
    - 계약번호 필드는 “저장 시 자동부여” 안내 문구와 함께 비활성화 상태로 표시된다.
6. 판매채널은 입력 완료 후 [저장] 버튼을 클릭한다.
7. 시스템은 입력된 정보의 필수 항목 누락 및 형식 오류 여부를 검증한다. **(E1,E2)**
    - 검증 항목 및 기준
        - 필수 항목 존재 여부: 이름, 주민등록번호, 연락처, 보험종류, 계약일, 만료일, 월보험료
        - 형식 검증: 주민등록번호(000000-0000000), 연락처(000-0000-0000), 계약일,만료일(YYYY.MM.DD)
        - 중복 검증: 동일 주민등록번호가 이미 존재하는지 확인한다.
8. 시스템은 고객번호와 계약번호를 자동으로 부여한다.
9. 시스템은 “고객 정보가 성공적으로 등록되었습니다” 라는 팝업 메시지를 출력한다.
10. 시스템은 고객 정보 등록 완료 화면으로 전환한다.
    - 고객 번호, 계약 번호가 부여된 고객 정보 화면이 출력된다.

**Alternative**

**A1) 입력 항목을 확인하고자 하는 경우,** 

1. 주민등록번호 우측 끝에 있는 [미리보기]버튼을 통해 확인할 수 있다.

---

**A2) 특약 정보를 추가 등록하고자 하는 경우,** 

1. 텍스트 필드 하단의 [추가] 버튼으로 항목 추가할 수 있다. 

**Exception**

**E1) 필수 항목이 누락되었거나 형식 오류가 발견된 경우**

1. 시스템은 해당 필드로 화면을 스크롤하고 오류 발생 필드에 빨간색 테두리를 표시한다.
2. 시스템은 필드 하단에 오류 안내 문구를 표시한다.
3. 입력된 나머지 항목의 값은 유지된다.
4. 판매채널은 오류 항목을 수정한 후 [저장]버튼을 클릭한다.
5. basic path 7번으로 돌아간다.

---

**E2) 중복 데이터가 감지된 경우**

1. 시스템은 화면 상단에 “EX. 이미 등록된 고객/계약번호입니다.”라는 중복 오류 메시지를 출력한다.
2. 시스템은 중복이 감지된 필드를 빨간색 테두리로 표시한다.
3. 판매채널은 중복 항목을 수정한 후 [저장]버튼을 클릭한다.
4. basic path 7번으로 돌아간다.

## 활동 계획을 작성한다

**Basic Path**

1. 판매채널은 메인 페이지 상단 메뉴바에서 [영업 활동] 메뉴 하위의 [활동 계획 작성] 항목을 클릭한다.
2. 시스템은 활동 계획 작성 화면을 출력한다.
    - 화면은 [기본 정보], [방문/상담 일정], [영업 목표], [제안 상품] 섹션으로 구성된다.
    - 화면 우측 상단에 [임시저장] 버튼과 [제출] 버튼이 배치된다.
3. 판매채널은 [기본 정보] 섹션에 계획의 기본 정보(계획명, 계획 기간, 메모)를 입력한다.
    - [기본 정보] 섹션에는 계획명, 계획기간, 작성자, 메모 필드가 배치된다.
    - 입력 항목(형식), 필수 여부
        - 계획명(텍스트), 필수
        - 계획기간(시작일~종료일, YYYY.MM.DD), 달력 팝업, 필수**(A1)**
        - 메모(텍스트), 선택
    - 작성자 필드는 로그인 계정으로 자동 입력되며, 비활성화 상태이다.
4. 판매채널은 [방문/상담 일정] 섹션에 고객별 활동 일정을 입력한다.
    - [방문/상담 일정] 섹션에는 [일정 추가], [삭제] 버튼이 배치된다.**(A2)**
    - [일정 추가] 버튼을 클릭하면 일정 입력 행이 추가된다.
    - 입력 항목(형식), 필수 여부
        - 대상 고객(고객 번호 입력), 필수
        - 활동 유형(방문/상담/전화), 드롭다운 선택, 필수
        - 활동 일시(YYYY.MM.DD HH:MM), 달력, 시간 팝업 제공, 필수
        - 활동 장소(텍스트), 선택
        - 메모(텍스트), 선택
    - 행 우측의 [삭제] 버튼으로 해당 일정 항목을 제거할 수 있다.
    - 일정은 활동 일시 기준 오름차순으로 정렬된다.
5. 판매채널은 [영업 목표] 섹션에 기간 내 달성 목표(계약 건수, 계약 금액, 신규 고객)를 입력한다.
    - [영업 목표] 섹션에는 목표 계약 건수, 목표 계약 금액, 목표 신규 고객 수 필드가 배치된다.
    - 입력 항목(형식), 필수 여부
        - 목표 계약 건수(숫자 입력, 단위: 건), 필수
        - 목표 계약 금액(숫자 입력, 단위: 원), 필수
        - 목표 신규 고객 수(숫자 입력, 단위: 명), 선택
6. 판매채널은 [제안 상품] 섹션에 고객에게 제안할 보험 상품 계획(대상 고객, 제안 보험 종류, 제안 사유)을 입력한다.
    - [제안 상품] 섹션에는 대상 고객, 제안 보험 종류, 제안 사유 필드가 배치된다.
    - 입력 항목(형식), 필수 여부
        - 대상 고객(고객 번호), 필수
        - 제안 보험 종류(생명/건강/자동차/화재), 드롭다운, 필수
        - 제안 사유(텍스트), 선택
7. 판매채널은 [제출] 버튼을 클릭한다.**(A3)**
8. 시스템은 입력된 정보의 필수 항목 누락 여부를 검증한다.**(E1)**
9. 시스템은 활동 계획을 저장하고 영업 관리자에게 검토 요청 알림을 발송한다.
10. 시스템은 화면 상단에 “활동 계획이 제출되었습니다. 영업 관리자의 검토 후 확정됩니다.”라는 팝업 메시지를 출력한다.
11. 해당 계획의 상태는 [검토 중]으로 표시된다.

**Alternative**

**A1) 종료일을 시작일보다 앞으로 설정할 경우,** 

1. 시스템은 “종료일은 시작일보다 이전일 수 없습니다. 다시 선택해주세요”라는 팝업 메시지를 출력한다.

**A2) 이미 일정이 작성되어 있는 경우**

1. 일정 행 하단에 [일정 추가] 버튼이 배치된다.

---

**A3) 임시저장이 필요한 경우**

1. 판매채널은 [임시저장]버튼을 클릭한다.
2. 시스템은 현재까지 입력된 내용을 임시저장 상태로 저장한다.
    - 필수 항목 누락 여부는 검증하지 않는다.
3. 시스템은 화면 상단에 “임시저장되었습니다”라는 메시지를 출력한다.
4. basic path 2번으로 돌아간다.

**Exception**

**E1) 필수 항목이 누락된 경우**

1. 시스템은 화면 상단에 “필수 항목을 확인해주세요”라는 팝업 메시지를 출력한다.
2. 시스템은 누락된 필드를 빨간색 테두리로 표시하고 하단에 “필수 항목입니다”라는 문구를 표시한다.
3. 입력된 나머지 항목의 값은 유지된다.
4. 판매채널 담당자는 누락 항목을 입력한 후 [제출] 버튼을 클릭한다.
5. basic path 8번으로 돌아간다.

## 문의한다

**Basic Path**

1. 고객은 메인 화면 상단 네비게이션 바에서 [고객센터] 버튼을 클릭한다.
2. 시스템은 고객 센터 페이지를 표시한다.
    - 페이지 상단에는 [1:1 문의], [자주 묻는 질문], [문의 내역 조회] 탭이 나열되어 있으며, 기본으로 [1:1 문의] 탭이 선택된 상태로 표시된다.
3. 고객은 [1:1 문의] 탭을 클릭한다.**(A1, A2)**
4. 시스템은 문의 작성 폼을 표시한다. 폼의 구성은 다음과 같다.
    - 문의 유형 선택 드롭다운(필수, 미선택 시 “선택해주세요” 안내 문구 표시)
    - 제목 입력란(필수, 최대 50자)
    - 내용 입력란(필수, 최대 1000자, 우측 하단에 현재 입력 글자 수/최대 글자 수 표시)
    - 파일 첨부 버튼(선택)
    - [제출] 버튼
    - 필수 항목은 항목명 우측에 빨간색 *로 표시된다.
5. 고객은 문의 유형 드롭다운을 클릭해서 보험료, 보험금, 계약 변경, 해지, 기타 중 하나를 선택한다.
6. 고객은 제목 입력란에 문의 제목을 입력한다.
7. 고객은 내용 입력란에 문의 내용을 입력한다. **(A3)**
8. 고객은 [제출] 버튼을 클릭한다.**(E1)**
9. 시스템은 입력된 문의 유형, 제목, 내용, 첨부 파일을 저장하고 고유한 문의 번호를 부여한다.
10. 시스템은 “문의가 정상적으로 접수되었습니다. 문의 번호: [문의 번호]”라는 메시지를 팝업 메시지로 출력한다. 팝업 메시지의 하단에는 [확인] 버튼이 있다.
11. 시스템은 고객의 연락처로 문의 접수 확인 문자를 발송한다. 문자에는 문의 번호, 문의 유형, 제목, 접수 일시가 포함된다.
12. 방금 접수된 문의가 문의 내역 목록 최상단에 ‘답변 대기’ 상태로 표시된다.
13. 고객은 해당 항목을 클릭한다. **(A4)**
14. 시스템은 문의 상세 페이지를 표시한다. 페이지는 문의 번호, 문의 유형, 제목, 내용, 첨부 파일(첨부한 경우), 접수 일시, 처리 상태(”답변 대기”)가 표시된다.

**Alternative**

**A1) [자주 묻는 질문] 탭을 클릭하는 경우**

1. 고객은 [자주 묻는 질문] 탭을 클릭한다.
2. 시스템은 FAQ 페이지를 표시한다.
    - 페이지 상단에는 카테고리 필터(전체, 보험료, 보험금, 계약 변경, 해지, 기타)가 나열되어 있으며, 기본값은 전체로 표시된다.
    - 페이지 하단에는 전체 FAQ 항목이 아코디언 형식으로 나열된다.
3. 고객은 카테고리 중 하나를 클릭한다.
4. 시스템은 선택한 카테고리에 해당하는 FAQ 항목만 필터링해 표시한다.
5. 고객은 FAQ 항목 중 하나를 클릭한다.
6. 시스템은 해당 항목의 답변 내용을 항목 하단에 펼쳐서 표시한다.

---

**A2) [문의 내역 조회] 탭을 클릭하는 경우**

1. 고객은 [문의 내역 조회] 탭을 클릭한다.
2. 시스템은 문의 내역 조회 페이지(페이지 소개)를 출력한다.
    - 페이지에는 고객이 접수한 문의 목록이 접수 일시 최신순으로 표시되며, 각 항목에는 문의 번호, 문의 유형, 제목, 접수 일시, 처리 상태(답변 대기/답변 완료)가 표시된다.
3. 고객은 확인하고자 하는 문의 항목을 클릭한다.
4. 시스템은 문의 상세 페이지를 표시한다.
    - 페이지에는 문의 번호, 문의 유형, 제목, 내용, 첨부 파일(첨부한 경우), 접수 일시, 처리 상태가 표시된다.

---

**A3) 파일을 첨부하는 경우**

1. 고객은 [파일 첨부] 버튼을 클릭한다.
2. 시스템은 파일 탐색기를 연다.
3. 고객은 첨부할 파일을 선택하고 [열기] 버튼을 클릭한다.**(E2)**
4. 시스템은 첨부 버튼 하단에 선택된 파일명과 파일 크기를 표시하고 파일명 우측에 파일 삭제를 위해 [x]버튼을 함께 표시한다.
5. basic path 8번으로 돌아간다.

---

**A4) 답변 완료된 문의를 확인하는 경우**

1. 고객은 처리 상태가 ‘답변 완료’인 문의 항목을 클릭한다.
2. 시스템은 문의 상세 페이지를 표시한다.
    - 페이지에는 문의 번호, 문의 유형, 제목, 내용, 첨부 파일(첨부한 경우), 접수 일시를 표시한다.
    - 페이지 하단에는 담당자의 답변 내용, 답변 일시가 표시된다.

**Exception**

**E1) 필수 항목을 입력하지 않았을 경우,**

1. 고객이 문의 유형, 제목, 내용 중 하나 이상을 입력하지 않은 상태에서 [제출] 버튼을 클릭한다.
2. 시스템은 제출을 처리하지 않고, 입력되지 않은 항목의 입력란 테두리를 빨간색으로 강조하고 해당 항목 하단에 “필수 입력 항목입니다.”라는 문구를 빨간색 텍스트로 표시한다.
3. 고객은 누락된 항목을 입력한다.
4. basic path 8로 돌아간다.

---

**E2) 첨부 파일이 허용 용량을 초과한 경우**

1. 고객이 선택한 파일의 용량이 10MB를 초과한다.
2. 시스템은 파일을 첨부하지 않고 “첨부 파일은 10MB 이하만 업로드 가능합니다.”라는 문구를 첨부 버튼 하단에 빨간색 텍스트로 표시한다.
3. 고객은 [파일 첨부] 버튼을 다시 클릭하여 다른 파일을 선택한다.
